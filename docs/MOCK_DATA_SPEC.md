# Спецификация моковых данных

Моковые данные нужны для разработки интерфейса до появления backend и базы данных.

## Категории

- Документы.
- ЖКХ.
- Налоги.
- Семья.
- Работа.
- Здоровье.
- Авто.
- Бизнес/ИП.

## Проблемы

- Потерял паспорт.
- Нужна медкнижка.
- Не пришла квитанция ЖКХ.
- Налог начислили неправильно.
- Переезд и регистрация.
- Рождение ребёнка.
- Увольнение.
- Открытие ИП.

## Документы

- Паспорт.
- Медкнижка.
- Автомобильные документы.
- ЖКХ/лицевой счёт.

## Уведомления

- Медкнижка истекает через 30 дней.
- Паспорт истекает через 120 дней.
- Передать показания ЖКХ до 25 числа.
- Проверить начисления налогов.

## Где хранить

Все моковые данные текущего этапа должны храниться в `src/data/mock_data.py`.

## Правила

- Не подключать реальную базу данных.
- Не подключать API.
- Не смешивать моковые данные с UI-компонентами.
- Если данных не хватает, добавить TODO или расширить моковые данные отдельной задачей.
