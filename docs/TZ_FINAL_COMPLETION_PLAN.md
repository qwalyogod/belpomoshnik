## 1. Текущий статус проекта

Проект находится на стадии первой беты. Основная пользовательская логика уже работает локально:

```text
жизненные сценарии
→ создание личной ситуации
→ задачи и документы
→ зависимости шагов
→ прогресс
→ дашборд
→ уведомления
→ профиль
→ админ-панель MVP
```

Рабочий стек сейчас:

- мобильный клиент: Flet/Python;
- локальное состояние: `data/app_state.json`;
- backup локального состояния: `data/app_state.backup.json`;
- backend-основа: FastAPI + SQLAlchemy + SQLite MVP;
- полноценные PostgreSQL, JWT, RBAC, SMTP и production-безопасность пока не реализованы.

До полного закрытия ТЗ проект нужно довести от локальной демонстрационной логики к более системной архитектуре: роли, полноценная админка, персонализация, уведомления, трекеры ЖКХ/налогов, API, безопасность и дипломная документация.

## 19. Критерий полного закрытия ТЗ

ТЗ можно считать закрытым на уровне дипломного MVP, когда:

- пользователь может выбрать сценарий и пройти его с зависимостями шагов;
- профиль влияет на подсказки, учреждения, закон-апдейты и уведомления;
- личные документы имеют сроки, маскирование и напоминания;
- ЖКХ и налоги ведутся как локальные трекеры;
