# K6. Административная панель

## 6.1 Назначение

Административная панель предназначена для редактора контента и администратора системы. Она предоставляет полный цикл управления контентом приложения без необходимости прямого редактирования базы данных или исходного кода.

Маршрут: `/admin`  
Файл: `src/pages/admin_page.py`  
Доступ: роли `editor` и `admin`.

## 6.2 Разделы панели

| Вкладка | Содержимое |
|---------|-----------|
| Сценарии | Список всех сценариев с возможностью создания, редактирования, удаления |
| Этапы | Этапы выбранного сценария, управление порядком |
| Шаги | Шаги выбранного этапа, зависимости, тип шага |
| Документы | CRUD шаблонов документов |
| Учреждения | CRUD государственных органов (адрес, телефон, сайт, режим работы) |
| Источники | Управление официальными источниками, статус проверки |
| Закон-апдейты | Публикация изменений законодательства |
| Аудит | Журнал действий редактора |

## 6.3 Управление сценариями (A1–A9)

### Создание сценария (A1)
- Поля: slug, название, описание, категория, сложность, ориентировочное время, теги.
- Привязка к проблеме.
- После сохранения — запись в `admin_audit_logs`.

### Редактирование сценария
- Все поля доступны для изменения.
- Поле `is_published` — переключатель публикации.

### Удаление с подтверждением (A2)
- Диалог подтверждения «Удалить сценарий X?».
- Каскадное удаление этапов и шагов.

### Управление этапами (A3–A4)
- Добавление / редактирование / удаление этапа.
- Поля: название, описание.

### Управление шагами (A5–A6)
- Добавление / редактирование / удаление шага.
- Поля: название, описание, тип шага (action / info / document / payment), обязательность.
- Привязка учреждения (Authority).
- Привязка дедлайна (Deadline).
- Ссылка на источник.

### Изменение порядка (A7)
- Кнопки «▲» / «▼» для изменения `order_index` этапов и шагов.
- Сортировка применяется немедленно.

### Зависимости шагов (A8)
- Визуальный список зависимостей: «Шаг X невозможен без шага Y».
- Добавление / удаление зависимостей через выпадающий список шагов.

### Связанные сценарии (A9)
- Поле для указания slug связанных сценариев (разделитель `,`).
- Тип связи: related / follow_up / alternative.

### Привязка источников (A10)
- К каждому сценарию / этапу / шагу можно привязать один или несколько источников.
- Поля: название, URL, дата проверки, статус.

## 6.4 CRUD учреждений (A11)

- Список всех госучреждений с поиском по названию и района.
- Создание: название, адрес, телефон, сайт, часы работы, район.
- Редактирование и удаление.

## 6.5 CRUD документов и шаблонов (A12)

- Список шаблонов документов.
- Поля: название, описание, флаг «является шаблоном», ссылка на скачивание.
- Привязка к шагам сценария.

## 6.6 Управление закон-апдейтами

- Создание закон-апдейта: заголовок, краткое описание, подробности, дата вступления в силу, источник.
- Привязка к категории профиля → автоматическое уведомление.
- Привязка к затронутому сценарию / шагу.
- Статус обработки: new → reviewed → applied.
- Дата проверки источника.

## 6.7 Журнал аудита (F4–F6)

- Хронологическая лента действий редактора.
- Каждая запись: действие, тип сущности, id, детали, дата.
- Фильтрация по типу действия (created / updated / deleted / published).
- Примеры действий:
  - `created_scenario` — создан сценарий «Регистрация автомобиля»
  - `updated_step` — изменён шаг 3 этапа «Подготовка документов»
  - `published_law_update` — опубликовано изменение «Новые ставки налога»

## 6.9 API endpoints для административных операций

```
GET    /api/admin/problems
POST   /api/admin/problems
PUT    /api/admin/problems/{id}

GET    /api/admin/scenarios
POST   /api/admin/scenarios
PUT    /api/admin/scenarios/{id}
DELETE /api/admin/scenarios/{id}

GET    /api/admin/scenarios/{id}/stages
POST   /api/admin/scenarios/{id}/stages
PUT    /api/admin/stages/{id}
DELETE /api/admin/stages/{id}

GET    /api/admin/stages/{id}/steps
POST   /api/admin/stages/{id}/steps
PUT    /api/admin/steps/{id}
DELETE /api/admin/steps/{id}

GET    /api/admin/documents
POST   /api/admin/documents
PUT    /api/admin/documents/{id}
DELETE /api/admin/documents/{id}

GET    /api/admin/authorities
POST   /api/admin/authorities
PUT    /api/admin/authorities/{id}
DELETE /api/admin/authorities/{id}

GET    /api/admin/law-updates
POST   /api/admin/law-updates
PUT    /api/admin/law-updates/{id}

GET    /api/admin/audit-log
GET    /api/admin/email-log
POST   /api/admin/email-queue/flush
```

Все эндпоинты защищены JWT + проверкой роли `editor` или `admin` через `Depends(require_editor)` / `Depends(require_admin)`.
