# K2. Предметная область

## 2.1 Жизненные ситуации гражданина

Предметная область — взаимодействие гражданина РБ с государственными органами. Выделены следующие типовые жизненные ситуации:

| # | Ситуация | Основные органы | Ключевые сроки |
|---|----------|----------------|----------------|
| 1 | Рождение ребёнка | ЗАГС, Минтруда, поликлиника | 3 месяца на оформление пособий |
| 2 | Увольнение с работы | Наниматель, ФСЗН, служба занятости | 2 недели отработка, 7 дней на постановку на учёт |
| 3 | Утеря паспорта | ОГиМ МВД, РУВД | 1 месяц на замену |
| 4 | Открытие ИП | ИМНС, ИП-регистрация через Едпортал | 5 рабочих дней |
| 5 | Регистрация автомобиля | ГАИ МВД | 10 дней после покупки |
| 6 | Алименты | Суд, нотариус, исполнительная служба | 3 года исковой давности |
| 7 | Покупка жилья | Нотариус, Регистрационная служба | 30 дней на регистрацию права |
| 8 | Пособия семьям с детьми | Минтруда, ФСЗН | 6 месяцев от даты рождения |

## 2.2 Ключевые понятия

**Проблема (Problem)** — тематическая область или жизненная ситуация верхнего уровня. Пример: «Документы», «Семья», «Работа». Проблема объединяет несколько сценариев.

**Сценарий (Scenario)** — пошаговая инструкция для конкретного случая (например, «Регистрация автомобиля»). Содержит этапы, шаги, документы, сроки, учреждения и источники.

**Этап (Stage)** — логический блок внутри сценария, объединяющий несколько шагов. Пример: «Подготовка документов», «Подача заявления».

**Шаг (Step)** — конкретное действие пользователя. Может иметь зависимости от других шагов (нельзя выполнить, пока не сделан предыдущий).

**Ситуация пользователя** — экземпляр сценария, запущенный конкретным пользователем. Хранит прогресс (какие шаги выполнены), заметки, дату начала.

**Задача (Task)** — шаг из сценария, переведённый в персональный список дел пользователя.

**Учреждение (Authority)** — государственный орган с адресом, режимом работы, телефоном, сайтом.

**Документ (Document)** — шаблон документа, который требуется для прохождения сценария. Может быть ссылкой на скачивание или текстовым описанием.

**Источник (Source Reference)** — ссылка на официальный нормативный акт или портал, подтверждающая корректность информации в шаге/сценарии.

**Закон-апдейт (Law Update)** — публикация об изменении в законодательстве, привязанная к категории профиля и затронутым сценариям. Пользователь получает уведомление.

## 2.3 Нормативная база

| Акт | Область |
|-----|---------|
| Кодекс РБ о браке и семье, ст. 91–104 | Алименты |
| Закон «О государственных пособиях семьям, воспитывающим детей» | Пособия при рождении |
| Трудовой кодекс РБ | Увольнение, больничный |
| Закон «О дорожном движении» | Регистрация ТС |
| Кодекс о земле, Жилищный кодекс | Покупка жилья |
| Налоговый кодекс РБ | Открытие ИП, налоги |
| Закон «О документировании населения» | Паспорта |

Источники проверены по состоянию на **май 2026 года**. Полный список — [`docs/SOURCES.md`](../SOURCES.md).

## 2.4 Роли пользователей системы

| Роль | Описание | Доступные функции |
|------|----------|------------------|
| `user` | Зарегистрированный гражданин | Просмотр каталога, запуск сценариев, ведение задач, трекеры, уведомления |
| `editor` | Редактор контента | Управление сценариями, шагами, документами, учреждениями, закон-апдейтами |
| `admin` | Администратор | Все права редактора + управление пользователями, ролями, просмотр аудита |
| (неавторизован) | Гость | Просмотр каталога проблем и сценариев без сохранения прогресса |

## 2.5 Бизнес-правила

1. Шаг нельзя отметить выполненным, если не выполнены его зависимости.
2. Уведомление о документе с истекающим сроком создаётся за настраиваемое количество дней (7 / 30 / 60 / 120).
3. ЖКХ-платёж считается просроченным, если текущая дата > дата плановой оплаты и статус ≠ «оплачено».
4. Редактор видит только контент, но не данные пользователей.
5. Все материалы приложения носят **справочный характер** и не являются юридической консультацией.
