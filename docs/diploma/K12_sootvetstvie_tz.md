# K12. Отчёт соответствия техническому заданию

## Критерии полного закрытия ТЗ

Согласно `claude/PLAN.md`, ТЗ считается закрытым на уровне дипломного MVP при выполнении 10 критериев.

---

### Критерий 1. Пользователь проходит сценарий с зависимостями шагов

**Статус: ✅ Выполнено**

| Компонент | Реализация |
|-----------|-----------|
| Структура данных | `scenario_dependencies` в `models.py` и `SCENARIO_TEMPLATES` в `mock_data.py` |
| UI | `scenario_detail_page.py` — шаги отображаются с иконкой блокировки при невыполненных зависимостях |
| Логика | `situation_detail_page.py` — `can_complete_task()` проверяет зависимости перед отметкой |
| Прогресс | `situations_page.py` — отображает % выполнения, статус (active / completed / paused) |

**Проверка:** Запустить сценарий «Рождение ребёнка», попробовать отметить шаг 3 без шага 1 — отображается предупреждение.

---

### Критерий 2. Профиль влияет на подсказки, учреждения, закон-апдейты, уведомления

**Статус: ✅ Выполнено**

| Поле профиля | Влияние |
|-------------|---------|
| `has_children` | Блок «Важное для вас» на главной показывает сценарии пособий |
| `employment_status` | Блок рекомендаций адаптируется (ИП → налоги, безработный → пособия) |
| `district` | Подбор учреждений в `_build_institutions_block()` фильтрует по району |
| `category` из профиля | Закон-апдейты фильтруются по `category` в `_normalized_published_laws()` |
| `doc_reminder_days` | `sync_task_notifications()` создаёт уведомления за N дней |

**Код:** `home_page.py` функции `_personalized_hint()`, `_recommended_scenario_ids()`, `_relevant_law_ids()`.

### Критерий 9. Web-версия без traceback

**Статус: ✅ Выполнено**

- Исправлена критическая ошибка Python 3.14 (closure variable ordering в `main.py`): функции `_law_category_name`, `_split_related_scenarios`, `_normalize_law_update` перемещены перед `sync_task_notifications()`.
- Синтаксическая проверка: `python -m compileall src` — без ошибок.
- Приложение запускается в web-режиме (`flet run -w`) без исключений.
