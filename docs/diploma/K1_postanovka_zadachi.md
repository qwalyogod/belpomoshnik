# K1. Постановка задачи

## 1.1 Актуальность

Граждане Республики Беларусь регулярно сталкиваются с необходимостью взаимодействия с государственными органами: регистрация актов гражданского состояния, оформление пособий, подача налоговых деклараций, получение справок и замена документов. Информация о порядке действий разрознена по сайтам министерств, правовых порталах и справочниках, что создаёт барьер для граждан, не имеющих юридической подготовки.

Существующие государственные порталы (portal.gov.by) ориентированы на оказание электронных услуг, но не предоставляют структурированных пошаговых инструкций на понятном языке с учётом жизненной ситуации конкретного пользователя. Коммерческие юридические сервисы требуют оплаты или предоставляют общие сведения.

## 1.2 Цель проекта

Разработать информационную систему «Белпомощник» — мобильно-десктопное приложение, которое предоставляет гражданам пошаговые инструкции для решения типовых жизненных ситуаций, связанных с государственными органами Республики Беларусь.

## 1.3 Задачи

1. Разработать каталог типовых жизненных ситуаций и проблем гражданина (рождение ребёнка, увольнение, утеря паспорта, открытие ИП, регистрация автомобиля и др.).
2. Реализовать механизм пошаговых сценариев с зависимостями между шагами, отслеживанием прогресса, сроками и ссылками на официальные источники.
3. Обеспечить персонализацию: профиль пользователя влияет на подбор учреждений, напоминания и подсказки.
4. Реализовать трекеры ЖКХ-платежей и налоговых обязательств.
5. Разработать систему уведомлений: истечение сроков документов, налоговые дедлайны, важные изменения законодательства.
6. Реализовать административную панель для редактора контента.
7. Спроектировать backend REST API с аутентификацией и разграничением прав доступа.
8. Задокументировать архитектуру системы и ограничения MVP.

## 1.4 Целевая аудитория

- Граждане РБ без юридической подготовки, впервые сталкивающиеся с административными процедурами.
- Родители при оформлении пособий и документов для детей.
- Индивидуальные предприниматели и самозанятые.
- Люди, изменившие жизненную ситуацию: смена работы, покупка жилья, развод, смерть близкого.

## 1.5 Границы разработки

В рамках дипломного MVP реализовано:
- Кросс-платформенное приложение на Flet (web + desktop + mobile layout).
- Полный цикл сценария от выбора ситуации до завершения всех шагов.
- Backend API на FastAPI + SQLAlchemy + SQLite с JWT-авторизацией.
- Административный интерфейс для управления контентом.
- Демонстрационная инфраструктура email-уведомлений (без реальной SMTP-отправки).

Вне границ MVP (описано в K7):
- Интеграция с порталом ЕГИАС или другими государственными API.
- Реальная отправка push- и email-уведомлений в production.
- Многопользовательский режим с облачной синхронизацией.

## 1.6 Ожидаемый результат

Программный продукт «Белпомощник», представляющий собой:
- Исходный код приложения и API.
- Набор контентных сценариев по 9 жизненным ситуациям.
- Дипломную документацию (K1–K12).
- Инструкции запуска и демонстрации.
