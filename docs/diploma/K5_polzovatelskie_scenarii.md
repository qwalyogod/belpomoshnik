## 5.2 Основные пользовательские сценарии (UX-flows)

### UC-01: Выбор ситуации и прохождение сценария

**Актор:** авторизованный пользователь  
**Предусловие:** пользователь вошёл в систему  
**Основной поток:**

1. На главной экране видит блок «Быстрые действия» → нажимает «Найти ситуацию».
2. Переходит на `/problems` — видит каталог проблем (14 тематических категорий).
3. Выбирает проблему «Семья» → открывается `/problems/family` с описанием и списком сценариев.
4. Нажимает «Начать сценарий» → открывается `/scenarios/childbirth`.
5. Просматривает этапы и шаги. Нажимает «Запустить сценарий».
6. Создаётся запись в `situations` с `status=in_progress`.
7. Переходит в `/situations` → видит созданную ситуацию.
8. Открывает `/situations/{id}` → список задач.
9. Отмечает задачу выполненной → шаг проверяет зависимости (следующая задача разблокируется).
10. После выполнения всех задач статус ситуации становится `completed`.

**Альтернативный поток:** шаг заблокирован зависимостью — отображается подсказка «Сначала выполните шаг X».

---

### UC-02: Управление личными документами

**Основной поток:**

1. Переходит на `/documents`.
2. Видит список документов с датами истечения срока.
3. Нажимает «Добавить документ» → диалог с полями: название, тип, дата истечения, заметки.
4. Загружает файл (PDF/JPG) — сохраняется в `data/private_uploads/`.
5. Документ с близкой датой истечения выделяется цветом.
6. Уведомление создаётся автоматически (за N дней до истечения).
7. Конфиденциальный документ помечается как скрытый — детали маскируются.

---

### UC-03: Трекер ЖКХ

**Основной поток:**

1. Переходит на `/utility`.
2. Добавляет лицевой счёт с адресом.
3. Вносит показания счётчиков и дату передачи.
4. Создаёт запись об оплате: сумма, дата плановой оплаты, статус.
5. При наступлении дедлайна — уведомление «Скоро нужно оплатить ЖКХ».
6. Просматривает журнал платежей.

---

### UC-04: Просмотр закон-апдейтов

**Основной поток:**

1. Переходит на `/legal-updates`.
2. Видит ленту изменений законодательства с фильтрацией по категории.
3. Важные изменения (matching профилю) выделяются.
4. Открывает детальную страницу → описание, дата вступления в силу, затронутые сценарии, источник.
5. Помечает закон-апдейт как прочитанный.
6. Уведомление о новом важном изменении приходит автоматически (если matching категории профиля).

---

### UC-05: Настройка профиля

**Основной поток:**

1. Переходит на `/profile`.
2. Заполняет: ФИО, дата рождения, район, адрес.
3. Устанавливает статус занятости (наёмный / ИП / студент / пенсионер / безработный).
4. Включает признаки: дети, автомобиль, собственник жилья.
5. Настраивает напоминания о документах (за 30 дней).
6. На главной появляются персонализированные подсказки.

---

### UC-06: Глобальный поиск

**Основной поток:**

1. Нажимает на иконку поиска в навигации или переходит на `/search`.
2. Вводит запрос (например, «паспорт»).
3. Результаты сгруппированы: Проблемы | Сценарии | Закон-апдейты | Документы.
4. Быстрые запросы (популярные теги) видны до ввода.
5. Нажимает на результат → переход на соответствующий экран.

---

### UC-07: Email-уведомление (демо)

**Основной поток:**

1. В `/notifications` видит уведомление типа `email_demo`.
2. Нажимает — открывается `/email-preview`.
3. Видит HTML-предпросмотр письма: заголовок, тело с истекающими документами, footer.
4. Видит баннер «MVP: письмо не отправляется. В production — SMTP через backend».
5. Нажимает «Отправить сейчас (demo)» → Snackbar «Отправлено (демо)».

## 5.3 Навигационная модель

### Desktop (ширина > 900px)
- Левый сайдбар: логотип, основные пункты меню, аватар пользователя.
- Основная область: контент экрана.
- Верхний AppBar с поиском и иконками уведомлений.

### Mobile (ширина ≤ 900px)
- Верхний AppBar с заголовком экрана и кнопками.
- Нижняя навигация (BottomNavigationBar): Главная, Сценарии, Ситуации, Документы, Профиль.

## 5.4 Персонализация

Профиль пользователя влияет на:

| Поле профиля | Влияние |
|-------------|---------|
| `district` | Подбор ближайших учреждений |
| `employment_status = "self_employed"` | Приоритет сценариев «Открытие ИП», «Налоги» |
| `has_children = True` | Показ пособий, сценарий «Рождение ребёнка» |
| `has_car = True` | Сценарий «Регистрация автомобиля» в рекомендациях |
| Категория профиля | Matching с закон-апдейтами и уведомлениями |
