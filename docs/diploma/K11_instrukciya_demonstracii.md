# K11. Инструкция демонстрации

> Данная инструкция описывает порядок демонстрации ключевых возможностей «Белпомощника» на защите дипломной работы.  
> Время демонстрации: 10–15 минут.

---

## Подготовка

1. Запустить приложение за 5 минут до начала:
   ```bash
   cd /Applications/XAMPP/xamppfiles/htdocs/belpomoshnik
   source .venv/bin/activate
   flet run -w src/main.py --host 127.0.0.1 -p 8550
   ```
2. Открыть браузер на **http://127.0.0.1:8550**.
3. Сбросить состояние если нужно (необязательно): `rm data/app_state.json`.
4. Приготовить второй экран или вкладку браузера с **http://127.0.0.1:8060/docs** (если backend запущен).

---

## Блок 1. Онбординг и авторизация (1 мин)

**Что показать:**
- Открыть `/onboarding` — приветственный экран.
- Перейти `/login` — ввести `user@example.by` / `123456`.
- Показать смену темы light ↔ dark (кнопка в профиле или в навигации).

**Ключевые тезисы:**
> «Приложение поддерживает обе цветовые темы. Авторизация реализована с JWT-токенами и ролевой моделью.»

---

## Блок 2. Главная и поиск (1–2 мин)

**Что показать:**
- Главная `/` — дашборд: приветствие с именем, персонализированные подсказки, быстрые действия.
- Блок истекающих документов (показывает паспорт с приближающимся сроком).
- Глобальный поиск `/search` — ввести «паспорт», показать сгруппированные результаты.

**Ключевые тезисы:**
> «Дашборд адаптируется под профиль. Глобальный поиск ищет одновременно по проблемам, сценариям и закон-апдейтам.»

---

## Блок 3. Каталог и сценарий (3–4 мин)

**Что показать:**
1. `/problems` — каталог из 14 тематических проблем.
2. Выбрать «Семья» → открыть `/problems/family`.
3. Нажать «Начать сценарий» для «Рождение ребёнка».
4. Показать страницу сценария `/scenarios/childbirth`:
   - Этапы: ЗАГС → Поликлиника → Пособия → ФСЗН.
   - Карточки шагов с документами, сроками, учреждением.
   - Зависимость шага (шаг 3 заблокирован до выполнения шага 1).
5. Нажать «Запустить сценарий».
6. Перейти в `/situations` — показать созданную ситуацию.
7. Открыть ситуацию, выполнить несколько задач.

**Ключевые тезисы:**
> «Сценарий разбит на этапы и шаги. Зависимости не позволяют пропустить обязательный шаг. Прогресс сохраняется.»

---

## Блок 4. Документы и уведомления (2 мин)

**Что показать:**
1. `/documents` — список личных документов с датами истечения.
2. Показать документ с красным маркером «истекает через 5 дней».
3. Добавить новый документ через диалог.
4. `/notifications` — список уведомлений.
5. Найти уведомление типа `email_demo` → нажать → открывается `/email-preview`.
6. Показать HTML-предпросмотр письма с баннером «MVP».

**Ключевые тезисы:**
> «Система автоматически создаёт уведомления о истекающих документах. Архитектура email-уведомлений готова к подключению SMTP.»

---

## Блок 5. ЖКХ и налоги (1–2 мин)

**Что показать:**
1. `/utility` — трекер ЖКХ: лицевые счета, показания, платежи.
2. Создать запись об оплате с приближающимся дедлайном.
3. `/taxes` — трекер налоговых обязательств.
4. Показать просроченное обязательство (выделено цветом).

**Ключевые тезисы:**
> «ЖКХ и налоговый трекеры работают локально и не требуют интеграции с реальными сервисами.»

---

## Блок 6. Закон-апдейты и профиль (1 мин)

**Что показать:**
1. `/legal-updates` — лента изменений законодательства.
2. Показать апдейт с привязкой к категории «Семья».
3. `/profile` — заполнение структурированных полей: статус занятости, дети, авто.
4. После сохранения — вернуться на главную, показать персонализированные подсказки.

---

## Блок 7. Административная панель (2–3 мин)

**Что показать:**
1. Переключиться в роль редактора: Profile → роль `content_editor`.
2. `/admin` — список сценариев.
3. Открыть сценарий «Регистрация автомобиля» → показать этапы и шаги.
4. Добавить новый шаг с полями (название, тип, учреждение).
5. Управление зависимостями: добавить зависимость между шагами.
6. Перейти на вкладку «Учреждения» → показать CRUD госорганов.
7. Перейти на вкладку «Аудит» → показать журнал с действиями редактора.

**Ключевые тезисы:**
> «Редактор может полностью управлять контентом без изменения кода. Все действия пишутся в audit log.»

---

## Блок 8. Backend API (1 мин, если запущен)

**Что показать:**
1. Открыть **http://127.0.0.1:8060/docs** — Swagger UI.
2. Показать группы эндпоинтов: public / admin / auth / user.
3. Выполнить `GET /api/problems` — вернётся список проблем в JSON.
4. Показать `POST /api/auth/login` с телом запроса.

**Ключевые тезисы:**
> «Backend реализован на FastAPI с JWT-авторизацией и RBAC. Интерфейс работает с mock-данными при недоступном API — graceful degradation.»

## Типичные вопросы комиссии и ответы

| Вопрос | Ответ |
|--------|-------|
| Почему не PostgreSQL? | SQLite достаточен для MVP. Переход — одна строка в DATABASE_URL. |
| Данные точные? | Источники проверены в мае 2026 г. Носят справочный характер. |
| Зачем Flet, а не React? | Единый Python-стек: UI + бизнес-логика + backend без JS. |
| Есть ли мобильное приложение? | Mobile layout работает в web. Нативная сборка — `flet build apk`. |
| Как масштабировать? | Backend на FastAPI горизонтально масштабируется за nginx. |
