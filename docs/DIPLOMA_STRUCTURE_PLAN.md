# План структуры дипломного проекта и пояснительной записки

Документ создан для проекта «Белпомощник» на основе методических указаний по выполнению дипломных проектов для специальности 2-40 01 01 «Программное обеспечение информационных технологий».

> Примечание: файл `documents/~$тодические указания ДП наше.doc` является временным lock-файлом Microsoft Word и не содержит полноценного текста методички. Основным источником для этого плана является файл `documents/Методические указания ДП наше.doc`.

## 1. Общая цель документа

Этот файл фиксирует, каким должен получиться дипломный проект в итоговом виде:

- из каких частей состоит дипломный проект;
- какие разделы должны быть в пояснительной записке;
- сколько страниц ориентировочно занимает каждый раздел;
- что именно писать в каждом разделе применительно к проекту «Белпомощник»;
- какие материалы вынести в приложения;
- какая графическая часть нужна;
- что должно быть в электронном проекте.

План нужен как рабочая карта перед написанием финальной версии диплома, чтобы не смешивать разделы, не терять требования методички и постепенно довести текст до нормального вида.

## 2. Исходные данные для титульного листа

Для текущего диплома использовать следующие данные:

| Поле | Значение |

Код `20.2.05` выбран как предварительный для информационной задачи. Если в официальном задании или у руководителя указан другой классификационный код, его нужно заменить в титульном листе и во всех местах документа.

## 3. Состав дипломного проекта

По методическим указаниям дипломный проект состоит из трёх крупных частей.

| Часть | Что входит | Для проекта «Белпомощник» |
|---|---|---|
| Пояснительная записка | Основной текстовый документ на 40-50 страниц печатного текста | Описание предметной области, проектирования, реализации, тестирования, экономики, охраны труда и выводов |
| Графическая часть | Обычно 2-4 листа формата A3, согласованные с руководителем | Схемы работы приложения, архитектуры, данных, пользовательского сценария и интерфейсных экранов |
| Электронный проект | Файлы проекта на электронном носителе | Исходный код Flet/Python, backend-основа, документация, README, презентация, пояснительная записка |

На защите нужно демонстрировать программный продукт. Если демонстрация невозможна технически, допускается презентация или видеофрагмент работы приложения.

## 4. Общие требования к оформлению пояснительной записки

| Параметр | Требование |
|---|---|
| Формат листа | A4, печать на одной стороне |
| Объём ПЗ | 40-50 страниц печатного текста |
| Шрифт | Times New Roman / Times New Roman Cyr |
| Размер основного текста | 14 пт |
| Межстрочный интервал | 18 пт |
| Выравнивание | по ширине |
| Цвет текста | чёрный, единый по всей работе |
| Поля | левое 30 мм, правое 10 мм, верхнее 20 мм, нижнее 20 мм |
| Переносы | включить автоматическую расстановку переносов |
| Содержание | формировать автоматически |
| Нумерация страниц | арабскими цифрами, по центру снизу, сквозная |
| Титульный лист и задание | входят в общий счёт страниц, номер на них не печатается |
| Новая глава | начинается с новой страницы |

Заголовки структурных частей `СОДЕРЖАНИЕ`, `ВВЕДЕНИЕ`, `ЗАКЛЮЧЕНИЕ`, `СПИСОК ИСПОЛЬЗОВАННЫХ ИСТОЧНИКОВ`, `ПРИЛОЖЕНИЯ` печатаются прописными буквами, по центру, полужирным шрифтом, на 1-2 пункта крупнее основного текста.

- надпись `ПОЯСНИТЕЛЬНАЯ ЗАПИСКА`;

- группа;
- руководитель;
- консультанты по экономике и охране труда;

Точную формулировку нужно сверить с приказом и заданием на дипломный проект.

### 6.2 Задание на дипломный проект

Задание оформляется на стандартном бланке. Оно должно содержать:

- тему проекта;
- исходные данные;
- перечень разделов пояснительной записки;
- перечень графической части;
- календарный график;
- подписи учащегося, руководителя, председателя цикловой комиссии и заместителя директора.

Для «Белпомощника» в задании желательно указать:

- разработку мобильного приложения на Flet/Python;
- локальное хранение данных;
- жизненные сценарии;
- пользовательские ситуации;
- документы;
- уведомления;
- обучающий модуль;
- админ-панель;
- подготовку к будущему backend/API.

### 6.3 Реферат

Объём: не более 1 страницы, рекомендуемый текст 850-1200 знаков.

Должен включать:

- сведения об объёме ПЗ;
- количество таблиц, рисунков, источников и приложений;
- ключевые слова;
- краткое описание цели и результата проекта.

Ключевые слова: 5-15 слов или словосочетаний, прописными буквами, через запятую, без точки в конце.

Примерный набор ключевых слов:

`МОБИЛЬНОЕ ПРИЛОЖЕНИЕ, FLET, PYTHON, ЖИЗНЕННЫЙ СЦЕНАРИЙ, ЛОКАЛЬНОЕ ХРАНЕНИЕ, УВЕДОМЛЕНИЯ, ДОКУМЕНТЫ, АДМИН-ПАНЕЛЬ, ИНФОРМАЦИОННАЯ ПОДДЕРЖКА`

### 6.4 Содержание

Содержание должно быть автоматическим.

В него должны попасть:

- перечень условных элементов;

- экономический раздел;
