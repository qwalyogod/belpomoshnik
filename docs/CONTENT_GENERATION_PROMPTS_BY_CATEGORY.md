# Промпты для генерации контента по категориям

Дата создания: 2026-06-09.

Этот документ нужен для получения реальных пачек контента через ChatGPT с
поиском в сети или Deep Research. В отличие от валидатора и JSON-контракта,
здесь описано не только **в каком формате вернуть данные**, но и **какой именно
контент искать** для каждой категории «Белпомощника».

## Как пользоваться

1. Открой ChatGPT.
2. Включи режим поиска в интернете / Deep Research.
3. Включи вывод источников, если такая настройка доступна.
4. Скопируй блок «Общая неизменяемая часть».
5. Ниже добавь один категорийный промпт, например «Документы».
6. Попроси вернуть только JSON без markdown.
7. Сохрани ответ в `data/import/content_<category>_2022_2026.json`.
8. Проверь файл:

```bash
.venv/bin/python scripts/validate_content_batch.py data/import/content_<category>_2022_2026.json
```

Если ChatGPT не может подтвердить факт официальной ссылкой, запись должна уйти
в `rejected_items`, а не превращаться в выдуманный материал.

## Общая неизменяемая часть

Эту часть вставлять в начало **каждого** промпта.

```text
Ты готовишь реальный контент для проекта «Белпомощник» — веб-системы для граждан Республики Беларусь.

Суть проекта:
«Белпомощник» собирает в одном месте информацию, которая уже размещена на официальных государственных сайтах, и объясняет её простым языком: проблема → понятный план действий → документы → сроки → учреждения → источники → напоминания.

Ты НЕ создаёшь юридическую консультацию от себя. Ты находишь официальные материалы, административные процедуры, новости ведомств, изменения законодательства и реестровые сведения, затем аккуратно пересказываешь их понятным языком для пользователя. Каждый юридически значимый факт должен иметь официальный источник.

Обязательно используй поиск в интернете или Deep Research.
Не используй только память модели.
Проверяй каждую ссылку перед добавлением в JSON.
Не выдумывай URL, даты, названия актов, номера документов, ведомства и сроки.
Если прямой официальный источник не найден, перенеси запись в rejected_items.

Период контента:
- с 2022-01-01;
- по 2026-06-08 включительно;
- не добавлять материалы после 2026-06-08.

Язык:
- весь пользовательский текст на русском языке;
- стиль простой, спокойный и понятный человеку 14-60+ лет;
- без канцелярита, но без искажения смысла официального источника.

Разрешённые официальные источники, уже заведённые в проекте:
- source-pravo — Национальный правовой Интернет-портал, https://pravo.by/
- source-portal-gov — Единый портал электронных услуг, https://portal.gov.by/
- source-mintrud — Министерство труда и социальной защиты, https://mintrud.gov.by/
- source-nalog — Министерство по налогам и сборам, https://nalog.gov.by/
- source-minjust — Министерство юстиции, https://minjust.gov.by/
- source-mvd — МВД / Департамент по гражданству и миграции, https://mvd.gov.by/ru/structure/departament/departament-grazhdanstva-i-migratsii
- source-gibdd — ГАИ МВД, https://mvd.gov.by/ru/structure/gai
- source-court — Верховный Суд Республики Беларусь, https://court.gov.by/
- source-nbki — сведения по недвижимости через госпортал, https://portal.gov.by/

Можно использовать и другие официальные сайты Республики Беларусь:
- сайты министерств;
- сайты комитетов;
- сайты облисполкомов, горисполкомов и райисполкомов;
- сайты служб «Одно окно»;
- сайты государственных организаций, если они являются первоисточником процедуры.

Если официальный источник нужен, но его нет в списке source_id, добавь его в source_candidates:
{
  "title": "Название источника",
  "url": "https://...",
  "type": "law|government_portal|ministry|tax|registry|court",
  "description": "Что подтверждает источник",
  "last_checked": "2026-06-08",
  "official": true,
  "reason_to_add": "Почему источник нужен проекту"
}

Медиа:
- если у официального источника есть подходящее публичное изображение, можно добавить URL;
- если официального изображения нет, оставь поля медиа пустыми;
- не используй стоки, соцсети, случайные СМИ, блоги и картинки из поиска;
- не добавляй изображение только ради красоты;
- если есть PDF, бланк, инструкция или официальный документ, можно добавить его в attachment_urls.

Нужно вернуть только один валидный JSON-объект, без markdown, без комментариев, без пояснений.

Корневой формат JSON:
{
  "batch_meta": {
    "project": "Белпомощник",
    "generated_for": "content_import",
    "category_id": "<CATEGORY_ID>",

    "checked_at": "2026-06-08",

  "rejected_items": []

Для категорийного батча подготовь:
- 12 problems;
- 5 scenarios;
- 18 news_articles;
- 12 law_updates.

Если подтверждённых материалов меньше, не выдумывай. Лучше верни меньше записей и добавь причины в rejected_items.

- news_article — это статья, а не короткая карточка;
- law_update — это статья о конкретном изменении, а не абстрактная справка;
- в конце каждого law_update обязательно должны быть блоки:
  <h3>Что изменилось</h3>
  <h3>Кого это касается</h3>
  <h3>Что сделать</h3>

Правила rejected_items:
Если запись выглядит полезной, но не подтверждена официально, положи её сюда:

  "searched_sources": ["URL или домен, где искал"],
  "notes": "Что нужно проверить вручную"

```

## Предложения по процессу

1. Генерировать не всё сразу, а по одному файлу на категорию.
2. После каждого ответа сразу запускать валидатор.
3. Сначала заполнить безопасные категории: `documents`, `family`, `work`, `business`.
4. Затем заполнить `housing`, `taxes`, `health`, `auto`.
5. Отдельно делать `news_calendar` и `law_updates`, потому что они самые объёмные.
6. `extremist_materials` делать последним и только с ручной проверкой каждой записи.
7. После успешной валидации писать импортёр в backend, но не раньше.
