# Промпты для наполнения контента «Белпомощника»

Дата обновления: 2026-06-08.

Документ нужен для ручной подготовки большого массива контента через ChatGPT или другой LLM с доступом к поиску. Цель - получить проверяемый JSON для дальнейшего импорта в backend проекта: категории, проблемы, жизненные сценарии, новости, закон-апдейты, официальные источники и раздел «Экстремистские материалы».

Важно: сгенерированный JSON не публикуется автоматически. Каждый URL и каждый юридически значимый факт должен проверить человек-редактор.

## 1. Как запускать генерацию в ChatGPT

Перед отправкой любого промпта включить:

- режим поиска в сети или Deep Research, если доступен;
- показ источников/цитирования, если доступен;
- запрет на использование только памяти модели;
- проверку дат публикаций и актуальности источников.

Если используется обычный чат без поиска, такие данные нельзя считать готовыми. В этом случае просить только структуру и список вопросов для проверки, но не фактический контент.

Общее правило для всех батчей:

- период публикаций и событий: с `2022-01-01` по `2026-06-08`;
- не создавать материалы позже `2026-06-08`;
- за один ответ просить не больше 40-60 записей, иначе JSON начинает ломаться;
- если источник не найден, запись отправлять в `rejected_items`, а не выдумывать ссылку;
- весь UI-текст на русском языке;
- пользовательские формулировки простые, без канцелярита;
- правовые утверждения только с официальной ссылкой.

## 2. Официальные источники, уже заведённые в проекте

Использовать эти `source_id`, если материал действительно опирается на соответствующий источник:

| `source-pravo` | Национальный правовой Интернет-портал | `https://pravo.by/` | law |
| `source-portal-gov` | Единый портал электронных услуг | `https://portal.gov.by/` | government_portal |
| `source-mintrud` | Министерство труда и социальной защиты | `https://mintrud.gov.by/` | ministry |
| `source-nalog` | Министерство по налогам и сборам | `https://nalog.gov.by/` | tax |
| `source-minjust` | Министерство юстиции | `https://minjust.gov.by/` | registry |
| `source-mvd` | МВД, Департамент по гражданству и миграции | `https://mvd.gov.by/ru/structure/departament/departament-grazhdanstva-i-migratsii` | government_portal |
| `source-gibdd` | ГАИ МВД | `https://mvd.gov.by/ru/structure/gai` | government_portal |

Если для темы нужен официальный источник, которого нет в таблице, ChatGPT должен вернуть его в `source_candidates`, а не использовать как готовый `source_id`.

## 3. Общий JSON-контракт для всех ответов

Просить ChatGPT возвращать только валидный JSON:

```json
{
  "batch_meta": {
    "project": "Белпомощник",
    "generated_for": "content_import",
    "category_id": "documents",
    "period_from": "2022-01-01",
    "period_to": "2026-06-08",
    "checked_at": "2026-06-08",
    "requires_human_review": true
  },
  "official_sources_used": [
    {
      "source_id": "source-pravo",
      "title": "Национальный правовой Интернет-портал",
      "url": "https://pravo.by/",
      "checked_at": "2026-06-08"
    }
  ],
  "problems": [],
  "scenarios": [],
  "news_articles": [],
  "law_updates": [],
  "extremist_entries": [],
  "source_candidates": [],
  "rejected_items": []
}
```

Поля, которые можно использовать в медиа:

```json
{
  "cover_url": "",
  "video_url": "",
  "gallery_urls": [],
  "attachment_urls": [],
  "media_notes": "Использованы только публичные официальные изображения или поле оставлено пустым."
}
```

Если официального изображения нет, `cover_url` оставлять пустым. Не брать случайные фото со стоков, соцсетей, новостных сайтов и блогов.

## 4. Базовый промпт для категории

Этот промпт используется для каждой категории. Ниже в разделе 5 есть готовые параметры для всех категорий.

```text
Ты готовишь контент для проекта «Белпомощник» - веб-системы для граждан Республики Беларусь.

Работай только с официальными источниками:
- pravo.by
- portal.gov.by
- mintrud.gov.by
- nalog.gov.by
- minjust.gov.by
- mvd.gov.by
- court.gov.by
- сайты государственных органов и ведомств Республики Беларусь

Обязательно используй поиск в сети или Deep Research. Не используй только память модели.
Проверяй каждую ссылку. Не выдумывай URL. Если прямой официальный источник не найден, перенеси запись в rejected_items.

Категория: <CATEGORY_ID>
Название категории: <CATEGORY_NAME>
Период: 2022-01-01 - 2026-06-08

Нужно подготовить JSON-батч:
1. 12 проблем для каталога.
2. 5 жизненных сценариев.
3. 18 редакционных новостей.
4. 12 закон-апдейтов.

Темы, которые нужно покрыть:
<CATEGORY_TOPICS>

Требования к problems:
- id: kebab-case на латинице;
- title;
- category: ровно <CATEGORY_ID>;
- shortDescription;
- whatToDoNow: 3-5 коротких действий;
- steps: массив шагов {id,title};
- documents: массив названий документов;
- deadlines: сроки, если подтверждены источником;
- institutions: типы учреждений;
- mistakes: частые ошибки;
- sourceIds: массив source_id из проекта;
- sources: массив {title,url,checked_at}.

Требования к scenarios:
- id;
- title;
- category: ровно <CATEGORY_ID>;
- shortDescription;
- longDescription;
- forWhom;
- estimatedTime;
- difficulty: "Простая" | "Средняя" | "Сложная";
- stages: этапы с задачами;
- tasks внутри stages должны иметь id, title, durationHint, dueOffsetDays или null, blockedBy;
- documents с обязательностью required true/false;
- institutions;
- sources;
- relatedIds.

Требования к news_articles:
- id;
- kind: "news";
- title;
- summary;
- bodyHtml: 4-7 абзацев полноценной статьи в HTML, допускаются <p>, <h3>, <ul>, <li>;
- cover;
- video;
- gallery;
- tags;
- category;
- source;
- sourceUrl;
- sourceIds;
- status: "published";
- date: дата между 2022-01-01 и 2026-06-08;
- media_notes.

Требования к law_updates:
- id;
- category: ровно <CATEGORY_ID>;
- title;
- summary;
- bodyHtml: полноценная статья, не просто справка. Сначала объясни событие и контекст, затем практическое значение.
- whoAffected;
- whatChanged;
- whatToDo;
- effectiveDate: не позже 2026-06-08;
- source: {id,title,url,description,checkedAt};
- sourceIds;
- priority;
- matchesProfile;
- media: {cover_url, gallery_urls, attachment_urls}.

В конце bodyHtml каждого law_update обязательно добавь блоки:
<h3>Что изменилось</h3>
<h3>Кого это касается</h3>
<h3>Что сделать</h3>

Верни только JSON по общему контракту из документа. Без markdown, без пояснений.
```

## 6. Промпт для большого календаря новостей 2022-2026

Этот промпт нужен, чтобы лента выглядела живой за несколько лет. Он не заменяет проверку фактов.

```text
Ты готовишь календарь редакционных публикаций для проекта «Белпомощник».
Используй поиск в сети или Deep Research.

Период: 2022-01-01 - 2026-06-08.
Нужно создать 120 новостных материалов, распределённых по годам:
- 2022: 24 материала;
- 2023: 28 материалов;
- 2024: 28 материалов;
- 2025: 28 материалов;
- 2026 до 8 июня: 12 материалов.

Категории должны быть распределены между:
documents, family, work, business, housing, taxes, health, auto.

Каждая новость должна быть связана минимум с одним официальным источником из списка проекта
или попасть в rejected_items, если прямого источника нет.

Формат каждой новости:
{
  "id": "news-...",
  "kind": "news",
  "title": "",
  "summary": "",
  "bodyHtml": "<p>...</p>",
  "cover": "",
  "video": "",
  "gallery": [],
  "tags": [],
  "category": "documents",
  "source": "",
  "sourceUrl": "",
  "sourceIds": ["source-pravo"],
  "status": "published",
  "date": "YYYY-MM-DD",
  "views_seed": 0,
  "media_notes": ""
}

Требования к bodyHtml:
- 4-7 абзацев;
- первый абзац отвечает, что произошло;
- второй объясняет, кого это касается;
- дальше практическое значение для пользователя;
- в конце абзац с указанием проверить сведения на официальном источнике;
- без клише, без рекламы, без эмоциональных заголовков.

Верни JSON по общему контракту. Заполняй только news_articles, остальные массивы оставь пустыми.
```

## 8. Промпт для экстремистских материалов

Раздел юридически чувствительный. Задача - не распространять содержание, а дать проверяемую выписку из официального реестра. Нельзя цитировать материалы, пересказывать идеологическое содержание, давать ссылки на неофициальные копии, добавлять изображения символики или скриншоты запрещённых материалов.

```text
Ты готовишь JSON для раздела «Экстремистские материалы» проекта «Белпомощник».
Работай только с официальными источниками Республики Беларусь.

Разрешённые источники:
- source-pravo, если запись связана с нормативным актом или решением;
- source-mvd, если информация опубликована МВД;
- source-minjust, если информация опубликована Министерством юстиции;
- source-court, если информация опубликована судом;
- source_candidates, если нужен официальный источник, которого ещё нет в проекте.

Не используй СМИ, Telegram, соцсети, блоги, зеркала, архивы и пересказы.
Не воспроизводи запрещённый материал. Не вставляй цитаты из материала.
Не добавляй фото/логотипы/скриншоты запрещённых ресурсов. Медиа допускается только как официальная иллюстрация страницы ведомства или пустое поле.

Период: 2022-01-01 - 2026-06-08.
Нужно подготовить 60 записей, если их можно подтвердить официальными источниками.
Если прямой официальный URL не найден, запись отправь в rejected_items.

Формат extremist_entries:
{
  "title": "Название записи ровно в нейтральной форме",
  "category": "registry",
  "source_url": "https://...",
  "source_name": "Официальный источник",
  "included_at": "YYYY-MM-DD или null",
  "last_checked_at": "YYYY-MM-DD",
  "short_description": "Нейтральное пояснение без цитирования и без оценки",
  "cover_url": "",
  "media_urls": [],
  "attachment_urls": [],
  "filters_json": "{\"content_types\":[\"media\"]}",
  "status": "draft",
  "verification_notes": "Что редактор должен сверить вручную"
}

Допустимые content_types:
- social;
- channels;
- media;
- persons;
- organizations;
- music;
- other.

Для новых публикационных батчей см. `docs/CONTENT_PUBLICATION_RESEARCH_PROMPTS.md`:
там допускается `status = "published"` для записей с прямым официальным
`source_url` и безопасным нейтральным описанием. Если уверенной проверки нет,
оставляй `status = "draft"` или переноси запись в `rejected_items`.

Верни JSON по общему контракту, заполнив extremist_entries.
```

## 9. Промпт для официальных источников и медиа

Использовать, когда нужно расширить каталог источников или подобрать медиа по ссылке.

```text
Ты валидируешь официальный источник или медиа-ссылку для проекта «Белпомощник».

Вход:
```

## 10. Что отдавать Codex для импорта

После генерации присылай мне один или несколько JSON-файлов. Лучше по батчам:

- `content_documents_2022_2026.json`;
- `content_family_2022_2026.json`;
- `content_work_2022_2026.json`;
- `content_business_2022_2026.json`;
- `content_housing_2022_2026.json`;
- `content_taxes_2022_2026.json`;
- `content_health_2022_2026.json`;
- `content_auto_2022_2026.json`;
- `content_news_calendar_2022_2026.json`;
- `content_law_updates_2022_2026.json`;
- `content_extremist_materials_2022_2026.json`.

Перед импортом я должен проверить:

- JSON парсится;
- нет дат позже `2026-06-08`;
- все `sourceIds` существуют в проекте или вынесены в `source_candidates`;
- у каждого факта есть официальный URL;
- у закон-апдейтов есть `bodyHtml`;
- у экстремистских материалов нет цитирования запрещённого содержания;
- медиа взяты только из официальных источников или оставлены пустыми.
