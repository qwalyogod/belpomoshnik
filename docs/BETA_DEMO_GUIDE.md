# Первая бета: сценарий демонстрации

Дата: 2026-05-19.

## Что такое первая бета

Первая бета «Белпомощника» — демонстрационная версия мобильного Flet-приложения, которую можно показать научному руководителю. Она показывает главный пользовательский путь: выбор жизненного сценария, создание личной ситуации, выполнение задач, контроль документов и прогресса.

Это не финальная версия диплома. Backend, полноценная авторизация, push-уведомления и публикация в магазинах приложений планируются следующими этапами.

## Что уже реализовано

- Главная страница как пользовательский дашборд.
- Каталог жизненных сценариев.
- Детальная страница сценария с этапами, задачами, документами, организациями и источниками.
- Создание пользовательской ситуации из шаблона.
- Задачи внутри ситуации с чекбоксами, сроками и документами.
- Автоматический расчёт прогресса ситуации.
- Локальное хранение в `data/app_state.json` и backup `data/app_state.backup.json`.
- Документы, уведомления, профиль и обучающий модуль.
- Базовая админ-панель как концепция будущего управления контентом.
- iPhone-ready layout с SafeArea для notch/Dynamic Island и home indicator.

## Как запустить

```bash
.venv/bin/flet run src/main.py
```

Для QR-кода и проверки на iPhone через Flet App:

```bash
.venv/bin/flet run --ios src/main.py
```

Для web-preview:

```bash
.venv/bin/flet run --web src/main.py
```

## Главный сценарий для показа

Основной сценарий демонстрации — «Рождение ребёнка». Он содержит 7 этапов: медицинские документы, документы родителей, регистрация рождения, свидетельство о рождении, пособия, регистрация по месту жительства и прикрепление к поликлинике.

## Демонстрационный путь

1. Открыть приложение и показать Главную.
2. Показать блоки дашборда: активные ситуации, ближайшие задачи, просрочки и документы.
3. Перейти в «Жизненные сценарии».
4. Открыть сценарий «Рождение ребёнка».
5. Показать описание, этапы, задачи, документы, организации и источники.
6. Нажать «Создать мою ситуацию».
7. Перейти в «Мои ситуации».
8. Открыть созданную ситуацию.
9. Отметить несколько задач выполненными.
10. Показать, что прогресс обновился.
11. Вернуться на Главную и показать обновлённый дашборд.
12. Открыть «Профиль» и показать достижения/обучение.
13. Открыть «Админ-панель» и показать концепцию наполнения сценариев.

## Что пока не реализовано

- Реальная backend-авторизация и роли.
- Полный переход мобильного приложения на API вместо локальных данных.
- PostgreSQL в production-режиме.
- Реальные push-уведомления.
- Реальная отправка email.
- Публикация в App Store / Google Play.
- Финальная юридическая выверка всех сценариев по официальным источникам.
