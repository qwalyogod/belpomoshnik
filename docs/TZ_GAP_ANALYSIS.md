# Gap-анализ соответствия техническому заданию

Дата анализа: 2026-05-19.

| Раздел ТЗ | Статус | Комментарий | Следующий шаг |
|---|---|---|---|
| Общая идея | Реализовано | Первая бета показывает главную идею: пользователь выбирает жизненный сценарий, создаёт личную ситуацию и проходит её по задачам | Расширить набор сценариев и проверить тексты по официальным источникам |
| Главная / У меня проблема | Реализовано | Главная работает как дашборд: быстрые действия, активные ситуации, ближайшие задачи, просрочки, документы и напоминания | После подключения API показывать данные пользователя с сервера |
| Каталог проблем | Реализовано в MVP | Есть каталог проблем, категории, поиск и фильтры на моковых данных | Подключить каталог к public API и расширить контент |
| Карточка проблемы | Реализовано в MVP | Есть пошаговый план, чекбоксы, документы, сроки, адресаты, ошибки и микро-тест | Сохранять прогресс карточки на сервере и юридически сверить данные |
| Life Events | Реализовано | Есть шаблоны сценариев, создание ситуации из шаблона, задачи, документы, сроки и автоматический прогресс | Перенести личные ситуации и задачи на backend |
| Мои документы | Частично реализовано | Есть локальные документы, статусы, добавление, редактирование и удаление | Добавить загрузку файлов, сроки, защищённое хранение и серверную синхронизацию |
| Уведомления | Частично реализовано | Есть локальный центр уведомлений и автогенерация по задачам с дедлайнами | Добавить cron/worker, push/email и серверные правила уведомлений |
| Закон-апдейты | Частично реализовано | Есть лента, фильтры, детальная страница и official URL в моковых данных | Реализовать реальный источник данных, проверку актуальности и персональную релевантность |
| Профиль | Частично реализовано | Есть локальный профиль, настройки, интересы, избранное, демо-сброс, обучение | Перенести профиль на backend и связать настройки с реальной авторизацией |
| Доступность | Частично реализовано | Интерфейс крупный и читаемый, есть переключатели крупного шрифта и высокого контраста | Применить настройки доступности глобально ко всем экранам и протестировать вручную |
| Админ-панель | Частично реализовано | Есть MVP UI и API для проблем, сценариев, этапов, шагов, документов и справочников | Довести UI CRUD для всех сущностей, добавить права редактора и аудит |
| Backend | Частично реализовано | Есть FastAPI + SQLAlchemy + SQLite MVP-основа и endpoints | Развить до production backend, закрепить API и подключить мобильный клиент |
| База данных | Частично реализовано | Есть SQLite MVP-база `data/belpomoshnik.db`, миграция и seed | Перейти на PostgreSQL и оформить миграции production-уровня |
| API | Частично реализовано | Есть public/admin endpoints, но мобильное приложение в основном остаётся на локальных данных | Перевести сценарии, ситуации, документы, уведомления и профиль на API |
| Авторизация | Частично реализовано | Есть локальный вход и регистрация для демо | Добавить backend-авторизацию, JWT, безопасное хранение паролей и восстановление доступа |
| Безопасность | Не реализовано | Нет production HTTPS, RBAC, bcrypt/argon2, аудита, политики хранения документов | Спроектировать security layer перед работой с реальными пользователями |
| MVP-критерии | Частично выполнено | Основной пользовательский путь и большинство MVP-экранов готовы; production-функции ещё впереди | Провести демонстрацию руководителю, зафиксировать замечания и перейти к backend/API этапу |
