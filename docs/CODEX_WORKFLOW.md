# Правила работы с Codex

## Перед задачей

- Читать `AGENTS.md`.
- Проверять `docs/PROJECT_STATUS.md`.
- Проверять `docs/TASKS.md`.
- Уточнять границы задачи.
- Если информации не хватает, писать `TODO`, а не выдумывать.

## Во время работы

- Не делать огромные изменения за один раз.
- Не менять архитектуру без причины.
- Не удалять файлы без явной необходимости.
- Не переписывать backend, базу данных, API или авторизацию без отдельной задачи.
- Сохранять русский язык интерфейса.
- Использовать Flet для экранов приложения.

## После изменения

- Запустить приложение, если менялся код Flet:

```bash
flet run src/main.py
```

- Если менялась только документация, проверить наличие и содержание Markdown-файлов.
- Обновить `docs/PROJECT_STATUS.md`.
- Обновить `docs/CHANGELOG.md`.
- Если есть ошибка, записать её в `docs/PROJECT_STATUS.md`.
- Если принято архитектурное решение, записать его в `docs/DECISIONS.md`.

## Проверка результата

- Приложение должно запускаться командой `flet run src/main.py`.
- Не заявлять как готовое то, что только планируется.
- Все новые ограничения и TODO фиксировать в документации.
