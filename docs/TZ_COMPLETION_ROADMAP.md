# Дорожная карта полного закрытия ТЗ

Дата фиксации: 2026-05-20.

Этот документ является рабочей дорожной картой развития проекта после первой демонстрационной беты. Его нужно читать перед крупными задачами по закрытию ТЗ вместе с `AGENTS.md`, `docs/PROJECT_STATUS.md` и `docs/TASKS.md`.

## Текущая логика проекта

Сейчас первая бета уже показывает основной пользовательский путь:

```text
сценарии → создание ситуации → задачи → документы → прогресс → дашборд
```

Следующая цель - довести систему до полного сценария ТЗ:

```text
пользователь + профиль + документы + адрес
→ персональные сценарии
→ связанные шаги
→ ближайшие госучреждения
→ уведомления
→ редактор контента
→ админ площадки
```

## Недостающие системные модули

- Роли `гражданин`, `редактор контента`, `администратор площадки`.
- Полноценная админ-панель и редактор контента.
- Расширенный профиль пользователя и персонализация.
- Личные документы, загрузка файлов и демонстрационная безопасность.
- Зависимости шагов внутри жизненных сценариев.
- Справочник госучреждений и подбор по адресу пользователя.
- Релевантные закон-апдейты.
- Email/push-уведомления и правила напоминаний.
- ЖКХ и налоги как упрощённые трекеры.
- Журнал действий пользователя и аудит админки.
- Production backend, PostgreSQL, RBAC, безопасность и резервное копирование.

## Правила движения

- Визуал не углублять без необходимости: сейчас фокус на внутренней логике и закрытии ТЗ.
- Не переписывать проект с нуля.
- Текущий мобильный клиент остаётся на Flet/Python.
- Текущий backend FastAPI + SQLite не заменять без отдельного решения.
- Новую логику сначала можно делать локально через `data/app_state.json`, если это ускоряет дипломный MVP.
- Backend/RBAC/PostgreSQL переносить отдельным крупным этапом после стабилизации локальной логики.
- Всё, что ещё не является production-готовым, помечать как демо/MVP/TODO.
