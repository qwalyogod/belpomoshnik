# Перенос дизайна из Figma во Flet

## Основные правила

- Не копировать HTML/CSS из Figma или сторонних генераторов.
- Flet-интерфейс строить через компоненты.
- Повторять структуру экранов из Figma максимально близко.
- Цвета брать из `docs/DESIGN_SYSTEM.md`.
- Не смешивать данные и UI.
- Не добавлять backend при переносе дизайна.

## Структура кода

- Каждый экран размещать в отдельном файле в `src/pages`.
- Общие элементы хранить в `src/components`.
- Нижнюю навигацию держать в отдельном компоненте.
- Карточки делать переиспользуемыми.
- Моковые данные хранить в `src/data/mock_data.py`.
- Цвета, отступы и константы хранить в `src/theme`.

## Навигация

- Навигацию делать через Flet-подход.
- Для развития проекта предпочтительно использовать `route_change`, если экранов станет больше.
- Нижняя навигация должна вести на основные разделы: Главная, Проблемы, Ситуации, Документы, Уведомления, Профиль.

## Проверка после переноса

- Приложение должно запускаться командой `flet run src/main.py`.
- Интерфейс должен оставаться на русском языке.
- Дизайн должен соответствовать Figma настолько близко, насколько позволяет Flet.
- Нельзя заявлять, что функциональность реализована, если перенесен только визуальный макет.
