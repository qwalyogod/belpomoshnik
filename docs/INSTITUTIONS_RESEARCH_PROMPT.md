# Промпт для сбора базы учреждений Республики Беларусь

Этот промпт нужен для ChatGPT с включённым поиском в сети или deep research. Цель — собрать справочник учреждений для проекта «Белпомощник», чтобы приложение могло показывать пользователю, куда обращаться по жизненной ситуации или проблеме с учётом региона, города и района.

## Как запускать

1. Включить web search / deep research.
2. Использовать только официальные источники: сайты исполкомов, министерств, ведомств, налоговых органов, МВД/ОГиМ, судов, органов соцзащиты, учреждений здравоохранения, служб «Одно окно», государственных порталов.
3. Не использовать агрегаторы, коммерческие каталоги, случайные карты и неофициальные справочники как основной источник.
4. Если данные не удалось подтвердить официальным источником, запись не включать в основной массив, а вынести в `rejected_items`.
5. Лучше делать отдельный запрос по каждой области/Минску, чтобы ответ не был слишком большим.

## Общие правила для модели

Ты собираешь данные для справочника учреждений веб-системы «Белпомощник» — гражданского помощника для жителей Республики Беларусь.

Не пиши обычный отчёт. Верни только валидный JSON по контракту ниже.

Требования к данным:

- не выдумывай адреса, телефоны, сайты и часы работы;
- каждое учреждение должно иметь минимум один официальный `source_url`;
- если телефон или часы работы не найдены на официальном источнике, оставь поле пустым;
- не добавляй персональные данные сотрудников;
- не добавляй частные компании, если они не выполняют государственную/административную функцию;
- для каждого учреждения укажи тип, регион, город/населённый пункт, район, адрес и услуги;
- если учреждение обслуживает несколько районов, укажи это в `services` и `tags`;
- поле `last_checked_at` поставь текущей датой проверки;
- поле `confidence` ставь:
  - `high`, если данные подтверждены официальной страницей самого учреждения/ведомства;
  - `medium`, если данные подтверждены официальным порталом или страницей исполкома;
  - `low`, если данные требуют ручной перепроверки, и такую запись лучше вынести в `rejected_items`.

## Типы учреждений

Используй только эти значения `institution_type`:

- `zags` — ЗАГС;
- `one_window` — служба «Одно окно»;
- `executive_committee` — исполком / администрация района;
- `citizenship_migration` — подразделение по гражданству и миграции / ОГиМ;
- `tax_office` — налоговая инспекция;
- `social_protection` — орган по труду, занятости и социальной защите;
- `healthcare` — поликлиника / учреждение здравоохранения;
- `court` — суд;
- `utility_service` — ЖКХ / РСЦ / расчётно-справочный центр;
- `traffic_police` — ГАИ / регистрационно-экзаменационное подразделение;
- `notary` — нотариальная контора / нотариальное бюро;
- `employment_center` — служба занятости;
- `education_department` — управление/отдел образования;
- `registry_real_estate` — организация по госрегистрации недвижимости;
- `other_state_service` — другое государственное учреждение.

## Промпт для одной области

Собери справочник государственных и административных учреждений для проекта «Белпомощник» по региону: `<ВСТАВИТЬ ОБЛАСТЬ ИЛИ МИНСК>`.

Нужно покрыть как минимум:

- областной центр;
- крупные города области;
- районные центры;
- службы «Одно окно»;
- ЗАГС;
- исполкомы и администрации районов;
- подразделения по гражданству и миграции;
- налоговые инспекции;
- органы соцзащиты;
- поликлиники/центральные районные больницы как точки прикрепления;
- суды;
- ЖКХ/РСЦ;
- ГАИ/регистрационно-экзаменационные подразделения;
- нотариальные конторы;
- службы занятости;
- отделы образования;
- организации по госрегистрации недвижимости.

Не нужно собирать абсолютно каждую маленькую организацию, если она не относится к административно-бытовым задачам гражданина. Приоритет — учреждения, которые нужны для сценариев:

- потеря паспорта;
- регистрация рождения ребёнка;
- пособия;
- регистрация по месту жительства;
- открытие ИП;
- налоги;
- ЖКХ;
- водительские документы и авто;
- медобслуживание;
- трудоустройство;
- жильё и недвижимость.

Верни только валидный JSON по контракту выше. Не добавляй Markdown, комментарии вне JSON или пояснительный текст после JSON.
