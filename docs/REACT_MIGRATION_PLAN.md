# План постепенной миграции на React/Vite frontend

Дата: 2026-06-03.

Этот документ описывает новый курс UI-развития проекта «Белпомощник»: основной пользовательский frontend постепенно переносится в `reactvitemaket/`, backend остаётся отдельным API, Flet UI временно сохраняется как рабочая версия и fallback.

## 1. Цель миграции

Получить красивый, адаптивный и дорогой по ощущению frontend, который проще развивать по Figma-дизайну, чем текущий Flet UI.

Целевая архитектура:

```text
frontend/ или reactvitemaket/
  Vite + React + TypeScript
  основной пользовательский интерфейс

src/backend/
  FastAPI + SQLAlchemy + SQLite MVP
  позже PostgreSQL, JWT, RBAC, email, аудит

shell_flet/ или legacy Flet
  опциональная оболочка/WebView или временный fallback

docs/
  требования, roadmap, отчёты, планы миграции
```

## 2. Главные правила миграции

- Не переписывать всё сразу.
- Не ломать рабочий Flet MVP, пока React не достиг parity.
- Backend FastAPI не заменять без отдельного решения.
- Не переносить моки хаотично, а строить API-контракты.
- Все новые React-экраны должны работать на desktop, tablet и mobile.
- Пользовательский UI должен быть на русском языке.
- Роли и protected actions должны повторять текущую логику проекта.
- Если функционал в React пока не подключён к API, помечать это как `TODO` в коде или документации.

## 6. Этап 3. Публичный контент

Статус: `IN_PROGRESS`.

Перенести на API:

- каталог проблем — начато через `/api/problems`;
- список жизненных сценариев — начато через `/api/scenarios`;
- деталь сценария — начато через `/api/scenarios/{slug}`;
- документы сценария — начато в составе full scenario detail;
- учреждения — подключены через `/api/authorities` и используются в глобальном поиске;
- официальные источники;
- справочник документов — подключён через `/api/documents` и используется в глобальном поиске;
- закон-апдейты — подключены через `/api/law-updates`.

Критерий готовности:

- пользователь без входа может открыть главную, каталог, сценарии, деталь сценария и закон-апдейты.
- глобальный поиск React-версии видит публичные документы и учреждения из backend API.
- одинаковые публичные сценарии из mock/backend не дублируются в каталоге React.
- backend summary отдаёт счётчики этапов и задач; React временно сохраняет богатые mock-массивы только для подробного preview, если summary не содержит этапы/документы/источники.

## 11. Этап 8. Упаковка приложения

Статус: `TODO`.

После React parity выбрать путь:

| Вариант | Когда подходит |
|---|---|
| PWA | быстрый web-first релиз и установка на устройства |
| Capacitor | iOS/Android как нативная оболочка для web UI |
| Tauri | desktop-приложение с лёгкой оболочкой |
| Electron | desktop с максимальной совместимостью, но тяжелее |
| Flet WebView-shell | только если нужно сохранить Python/Flet как тонкую оболочку |

Предварительная рекомендация:

- для web и mobile: PWA + Capacitor;
- для desktop: Tauri;
- Flet WebView использовать только как временную или учебную оболочку, если это потребуется для дипломной демонстрации.
