# Сценарий демонстрации к защите диплома

Дата: 2026-06-11. Тайминг: 5–7 минут.

## Цель

Показать, что «Белпомощник» — цельный мобильный/веб-MVP: пользователь описывает проблему, получает пошаговый план, может сохранить его, отмечать шаги, получать напоминания и видеть релевантные изменения законодательства. Второй сценарий — редактор публикует закон-апдейт, и гражданин видит его как «для меня».

## Подготовка (до выступления)

1. Запустить backend: `PYTHONPATH=src .venv/bin/uvicorn backend.app:app --host 127.0.0.1 --port 8060`
2. Запустить frontend: `cd reactvitemaket && pnpm dev`
3. В `BELPOMOSHNIK_ENABLE_TEST_SWITCHER=true` (в `.env`) — откройте логин как `editor@test.local` / `Test12345!` (для редакторского сценария).
4. Открыть `http://127.0.0.1:8560/` в браузере. В шапке WelcomePage виден бэйдж «Backend» (зелёный) — это подтверждает, что UI тянет данные из FastAPI.

## Быстрый показ админ-панели

1. Войти как администратор.
2. Открыть «Админ-панель» → «Обзор» и показать реальные KPI: пользователи, публикации, проблемы, сценарии, учреждения.
3. Открыть «Проблемы» и показать, что все проблемы из каталога видны в админке, включая статус и категорию.
4. Открыть «Сценарии» и показать количество этапов/задач, фильтры и поиск.
5. Открыть «Пользователи и роли»: изменить роль тестового пользователя, заблокировать/разблокировать, отозвать сессии.
6. Открыть «Журнал действий» и показать записи по изменению ролей, блокировкам и контентным операциям.
7. Открыть «Регионы и города»: выбрать регион на карте, показать правую панель, переименовать регион и сохранить. Карточка остаётся на прежнем месте.
8. Включить режим «Расположение», перетащить карточку региона и сохранить расположение. Координаты сохраняются в процентах относительно карты.

## Скрытый Control Center владельца

1. Открыть консоль браузера и выполнить `belpomoshnikControl()`.
2. Ввести пароль владельца платформы.
3. Показать, что панель не является пунктом меню и не назначается как роль пользователю.
4. Открыть «Статус»: backend, база, пользователи, публикации, проблемы, сценарии, учреждения.
5. Включить системный баннер, сохранить и показать, что он появляется поверх сайта.
6. Открыть «Режимы»: показать техническое обслуживание и режим только чтения, но не оставлять их включёнными после показа.
7. Открыть «Уведомления»: отправить системное in-app уведомление тестовым пользователям.
8. Открыть «Журнал»: показать, что действия Control Center логируются отдельно.

## 1. Гость ищет проблему (60 сек)

- Открыть `/welcome` (по умолчанию редирект с `/` для гостя).
- Проговорить: «Справа в шапке — индикатор Backend. Это значит, что контент идёт из FastAPI, а не из встроенных mock-данных».
- В блоке «Что случилось?» кликнуть «Потерял паспорт» → открывается `ProblemDetailPage`.
- Показать структуру: «Что делать прямо сейчас» (амбер-блок, sticky на мобиле), пошаговый план с прогресс-баром, документы, сроки, куда обращаться, частые ошибки, **источник/проверено** (Pill «Черновик»).
- Кнопка «Сохранить в избранное» работает (звёздочка), «Создать персональный план» → редирект на `/login?reason=create-plan` (GuestGuard).

## 2. Регистрация и сохранение (45 сек)

- На `/login` нажать «Создать аккаунт».
- Заполнить имя, email, пароль (≥ 6), регион. Submit.
- UI возвращается на ту же страницу /welcome (или /catalog). В шапке — аватар пользователя.
- Повторить клик на «Потерял паспорт» → «Создать персональный план» теперь работает: переход на `/catalog?from=...` со сценарием.

## 3. Моя ситуация и отметка шагов (60 сек)

- Открыть `/scenarios` (Каталог) → выбрать «Рождение ребёнка».
- Нажать «Создать мою ситуацию» → новая ситуация в `/situations`.
- Прогресс-бар 0%. Чекбоксы шагов с lock-иконкой для заблокированных.
- Отметить 2 шага → прогресс 28%, видно изменение в дашборде.
- В блоке «Ближайший срок» появился дедлайн из `buildReminders()`.

## 4. Уведомления и навигация (45 сек)

- Перейти на `/notifications`.
- Показать блок «Способы уведомлений»: внутренний центр включён всегда, browser/native каналы требуют разрешения пользователя.
- Нажать «Отправить тестовое»: сообщение сразу появляется в центре. Без разрешения ОС оно не всплывает, что является ожидаемым поведением.
- Показать, что **карточки теперь кликабельны** (новое): клик по «Проверьте документ» → `/documents`, клик по «Шаг выполнен» → `/situations`.
- Кнопка «Прочитать все» снимает флаг `read`, уведомления становятся opacity-60.

## 5. Документы: добавление и защищённый скан (45 сек)

- Открыть `/documents`.
- Кликнуть «+ Добавить документ» → выбрать тип «Паспорт», ввести номер, срок действия.
- Карточка появляется в списке, с цветной полоской (зелёная/жёлтая/красная) по оставшемуся сроку.
- **Проговорить**: «Чувствительные поля и сканы шифруются Fernet. Сканы выдаются только владельцу через JWT endpoint `/api/user/documents/{id}/scan` и не находятся в публичной static-папке».

## 6. Лента новостей и фильтр «для меня» (45 сек)

- Открыть `/news`.
- Переключатель «Все / Новости / Закон-апдейт» + фильтр по источнику.
- Карточка «Источник: pravo.by · проверено: 2026-05-12» — видна meta-информация.
- Клик на новость → детальная страница с `bodyHtml` (санитизированным) + секции «Что изменилось / Кого касается / Что сделать».

## 7. Редактор: публикация закон-апдейта (60 сек)

- Sign out → войти как `editor@test.local` / `Test12345!`.
- Перейти на `/editor` → вкладка «Закон-апдейты».
- Создать новый апдейт: категория «ЖКХ», заголовок, whatChanged, whatToDo, приоритет HIGH, источник, опубликовать.
- В audit-log (`/editor` → вкладка «Аудит») появилась запись «Создан закон-апдейт».
- Sign out, войти как citizen (или новый пользователь) → на `/news` новый закон-апдейт виден в ленте.

## 8. Архитектура и стек (30 сек)

- Сказать:
  - Backend: FastAPI 0.116 + SQLAlchemy 2.0 + Pydantic 2.11. MySQL dev/MVP, PostgreSQL-ready.
  - Frontend: React 18 + Vite 6 + TS + Tailwind 4 + motion, react-router 7. Mobile + desktop shell.
  - Native: Capacitor 8 → APK/IPA. Flet 0.85 — только WebView-обёртка (`src/mobile_webview.py`).
  - backend покрыт targeted-тестами; для текущего показа достаточно production frontend build и backend compile/import check.
- Показать команды проверки, которые запускались перед демонстрацией: `pnpm -C reactvitemaket run build`, `.venv/bin/python -m compileall -q src`.

## 9. Что НЕ реализовано (честный ответ на вопросы)

- Refresh-токены в БД хранятся в открытом виде (MVP). SHA-256 — production TODO.
- Login rate-limit in-memory (сбрасывается при рестарте).
- Document scan ACL — endpoint требует JWT и owner-check; сканы лежат вне публичной `/uploads`.
- Полная HTML-санитизация (DOMPurify/bleach) — сейчас только regex.
- In-app и локальные уведомления работают; серверная native push доставка потребует production credentials FCM/APNs.
- Реальная почтовая доставка: SMTP-очередь есть, но без `SMTP_*` env-ов email'ы зависают в `email_notifications`.
- Feature flags и navigation layout уже сохраняются через Control Center, но не все разделы ещё полностью отключаются/переупорядочиваются на runtime.

Все эти пункты — в `docs/PROJECT_STATUS.md` → «Сознательно отложено в Перспективы».
