# Редакторская панель и пользовательский контент — целевое видение

Дата фиксации: 2026-06-05. Источник: ТЗ пользователя (с референс-скриншотами CMS).

Это итоговое видение. Строится поэтапно на базе уже существующей плавающей админ-панели
(`reactvitemaket/src/app/components/desktop.tsx` AdminPanel + `admin-window.tsx`), в стилистике проекта.
Доступ к редакторской: роль `editor` И `admin`.

## 1. Редакторская панель НЕ открывается сразу в редактор

- При открытии редакторской первым показывается **экран аналитики/отчётов** (приветствие + аналитика), а не форма статьи.
- Редактор видит: графики, общее число просмотров за период, просмотры по каждой статье/проблеме/ситуации.
- Цель: редактору приятнее работать, когда он видит, что его материалы интересны.
- Сам редактор контента (форма) открывается уже по действию «Новая публикация» / «Изменить».

## 2. Переключатель типа контента

- Не только новости. Редактор переключается между: **Новости, Ситуации (сценарии), Проблемы** и т.д.
- Один универсальный редактор умеет создавать любой тип контента платформы.

## 3. Универсальный редактор контента (как на скриншотах)

Вкладки: **Контент / Медиа / Публикация** + «Предпросмотр».

- **Контент**: Заголовок, Краткое описание (для карточки и поиска), Текст (абзацы = пустая строка), Теги.
  - Добавить: панель быстрого форматирования — **жирный, курсив, подчёркнутый**.
  - Добавить: выделить слово/фрагмент, нажать кнопку → вставить **гиперссылку**.
- **Медиа**: Обложка (upload или URL), Видео (upload или URL), Лента фото для слайдера.
- **Публикация**: Категория, Специализация, Аудитория, Автор, Дата публикации, «Видно гостям»,
  Источник + ссылка на источник, Текст для гостя / Текст после входа.
- Кнопки: **Опубликовать**, **В черновик**, **Отмена**. Индикатор готовности (Заголовок · Текст · Обложка).

## 4. Пользовательские предложения контента (UGC)

- Кнопка для обычных пользователей — НЕ в меню, а на страницах **Новостей, Ситуаций, Проблем**:
  «Предложить свою ситуацию / новость / проблему».
- Пользователь создаёт контент через тот же упрощённый редактор → контент уходит в
  **очередь модерации** редактора.
- Редактор в очереди модерации может: **опубликовать, отредактировать, отклонить**.

## 5. Авторство (везде)

- На каждой статье/новости/ситуации/проблеме показывается автор:
  - Редактор создал: «Автор: Имя Фамилия».
  - Предложено пользователем: «Предложено пользователем: Имя Фамилия».
  - Редактор доработал предложение пользователя: «Предложено пользователем: «пользователь» при поддержке «автор контента»».

## 6. Анонимность пользователя

- Чекбокс «Анонимно» при отправке. Тогда имя автора скрыто публично.
- НО редактор всё равно видит, кто отправил (для модерации, борьбы со спамом).
- При спаме редактор может пожаловаться администрации, админ может заблокировать пользователю право предлагать контент.

## 7. Черновики

- Черновики для редакторов И для пользователей: локально сохранить материал, дописать позже.

## 8. Редактирование после публикации

- После публикации только редактор/админ может менять статью.
- Кнопка «Изменить» на статье видна только редактору и админу.
- По «Изменить» открывается редактор со всеми полями, заполненными этой статьёй; всё можно менять.
- Вместо «Опубликовать»/«В черновик» в режиме редактирования — кнопка «Изменить» (сохранить изменения).

## Порядок реализации (предлагаемый)

1. Редакторская = аналитический лендинг (приветствие + отчёты, данные пока производные/представительные).
2. Универсальный редактор контента (Контент/Медиа/Публикация, rich-text toolbar bold/italic/underline/link, теги).
3. Переключатель типа контента (Новости / Ситуации / Проблемы).
4. Авторство в моделях + отображение на карточках/деталях.
5. Черновики (локально, потом backend).
6. UGC: кнопки «Предложить…» на страницах + очередь модерации + публикация/правка/отклонение.
7. Анонимность + жалоба/блок пользователя.
8. Backend: модели публикаций, медиа, авторство, модерация, аналитика просмотров (production-этап).

## Ограничения сейчас
- Аналитика просмотров: реального трекинга нет, используем производные/представительные метрики, помеченные как пример, до backend-трекинга.
- Текущий редактор один (тестовый), но модель авторства multi-editor.

## Статус реализации (обновлено 2026-06-05)
Фронтенд E1–E8 готов и проверен end-to-end (local-first, без backend):
- E1 ✓ Редакторская = аналитический лендинг (Обзор редакции: приветствие, KPI, график, топ материалов, период 7/30).
- E2 ✓ Универсальный редактор (`content-editor.tsx`): Контент/Медиа/Публикация + Предпросмотр, rich-text (Ж/К/Подчёркнутый + ссылка), теги, готовность.
- E3 ✓ Переключатель типа (Новости/Ситуации/Проблемы) в редакторе + фильтр в Публикациях.
- E4 ✓ Авторство: `Автор: …` / `Предложено: … · при поддержке …` / аноним → `Аноним` (`authorLabel`).
- E5 ✓ Черновики: редактор (Публикации, статус draft) + пользователь («В черновик» в propose, «Мои предложения» в профиле → «Продолжить»).
- E6 ✓ UGC: кнопки «Предложить…» на Каталоге/Проблемах/Новостях → раздел «Модерация» (Опубликовать/Изменить/Отклонить).
- E7 ✓ Анонимность (публично скрыто, редактор видит отправителя) + «Пожаловаться» (report) + админ «Заблокировать» + enforcement.
- E8 ✓ Правка после публикации: «Изменить» открывает редактор с предзаполнением → «Сохранить изменения».

Данные: глобальный store `belp.articles` (общий для всех ролей, чтобы модерация видела заявки граждан) + `belp.blockedSubmitters`.
Типы `Article`/`AuthorMeta` в `types.ts`; слайс + действия в `store.tsx`.
