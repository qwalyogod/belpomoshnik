## Известные проблемы

- Глобальная команда `flet` может быть недоступна без активации `.venv`.
- Команды `python` и `pip` в системе могут отсутствовать; использовать `python3`, `pip3` или команды из `.venv`.
- Поиск и фильтрация реализованы только на моковых данных.
- Авторизация реализована как локальный MVP без реальной безопасности, JWT и RBAC.
- Для работы админ-функций Flet-экрана backend API должен быть запущен отдельно (`uvicorn backend.app:app ...`).
- Обучающий модуль реализован как демо/MVP на локальном состоянии без backend.
- Backend добавлен как MVP-основа, но мобильный клиент переведен на API только частично (пока «Рождение ребёнка»).
- В конструкторе сценариев пока нет drag-and-drop сортировки; порядок задается числом `order_index`.
- В админке пока нет удаления сущностей и массовых операций.
- Локальные пользовательские данные сохраняются в `data/app_state.json`, но это не заменяет backend-хранилище и реальную безопасность.
- Личные документы в демо-версии хранятся локально без шифрования; это допустимо для MVP, но не для production.
- На некоторых платформах Flet может не отдать локальный путь выбранного файла; в этом случае поле «Файл / скан» можно заполнить вручную для демо.
- Справочник учреждений демонстрационный: адреса и телефоны требуют финальной проверки по официальным источникам перед production.
- Шаблоны сценариев пока моковые; позже их нужно получать из public API сценариев.
- `flet run src/main.py` запускает долгоживущий GUI-процесс; для остановки использовать `Ctrl+C`.
- При запуске `flet run -w ...` важно использовать свободный порт; если порт занят, нужно выбрать другой.
- Android-адаптация и планшетные раскладки не выполнялись в текущем iPhone-этапе.
- На реальном iPhone нужно дополнительно проверить поведение клавиатуры в диалогах редактирования документов, ситуаций и задач.

## Команды запуска

С активацией окружения:

```bash
source .venv/bin/activate
flet run src/main.py
```

Без активации окружения:

```bash
.venv/bin/flet run src/main.py
```

Запуск в браузере для проверки внешнего вида:

```bash
.venv/bin/flet run -w src/main.py --host 127.0.0.1 -p 8550
```

Запуск на подключенное iOS-устройство, если Flet CLI и устройство уже настроены:

```bash
.venv/bin/flet run --ios src/main.py
```

Инициализация БД сценариев:

```bash
PYTHONPATH=src python -m backend.scripts.init_db
PYTHONPATH=src python -m backend.scripts.seed_db
```

Запуск API:

```bash
PYTHONPATH=src uvicorn backend.app:app --host 127.0.0.1 --port 8060
```

## Важные ограничения

- Не переписывать backend-стек (FastAPI + SQLAlchemy + SQLite MVP) без отдельной задачи.
- Не добавлять полноценную авторизацию и RBAC без отдельной задачи.
- Мобильный UI пока в основном работает на моковых данных; API подключается поэтапно.
- Все пользовательские тексты интерфейса писать на русском языке.
- После крупных изменений обновлять этот файл.
