# Промпты для наполнения «Белпомощника» данными

Дай эти промпты ChatGPT (лучше с веб-поиском: GPT-4o / o3 с browsing).
Каждый промпт выдаёт **готовый JSON**. Сохрани результат в файл с указанным именем
в папку `data/import/`. Когда все файлы будут на месте — скажи мне «импортируй данные»,
и я напишу загрузчик, который вольёт их в проект и создаст «видимость» давно живущей системы.

---

## Куда класть результаты

```
data/import/
  ├── institutions.json     ← Промпт 2 (госучреждения)
  ├── problems.json         ← Промпт 3 (каталог проблем)
  ├── scenarios.json        ← Промпт 4 (пошаговые сценарии)
  └── news.json             ← Промпт 5 (новости / закон-апдейты / реестр)
```

Регионы/города (Промпт 1) не сохраняются отдельным файлом — они нужны как
справочник внутри `institutions.json`. Но если хочешь — сохрани как
`data/import/regions.json`.

**Формат файлов:** UTF-8, валидный JSON-массив (начинается с `[`, заканчивается `]`).
Без markdown-обёрток ```json — только чистый JSON внутри файла.

## ПРОМПТ 1 — Регионы, города и райцентры Беларуси

```
Ты — справочник административно-территориального деления Республики Беларусь.

Выдай полный список в формате JSON-массива. Структура каждого объекта:
{
  "region": "<название области ИЛИ 'г. Минск'>",
  "region_center": "<областной центр>",
  "districts": [
    {"name": "<район>", "center": "<город/посёлок — райцентр>"}
  ]
}

Включи все 6 областей + город Минск:
- г. Минск (с 9 районами города)
- Минская область (центр Минск, все районы)
- Брестская область (центр Брест)
- Витебская область (центр Витебск)
- Гомельская область (центр Гомель)
- Гродненская область (центр Гродно)
- Могилёвская область (центр Могилёв)

Для каждой области перечисли ВСЕ административные районы и их центры.
Только реальные названия. Выведи чистый JSON-массив без пояснений.
```

Сохрани как: **`data/import/regions.json`**

## ПРОМПТ 2 — Государственные учреждения (имя + адрес + контакты)

```
Ты — справочник государственных учреждений Республики Беларусь.

Для каждого областного центра (Минск, Брест, Витебск, Гомель, Гродно, Могилёв)
и для 5-7 крупных райцентров каждой области собери реальные госучреждения
следующих типов:
- ЗАГС
- Исполком (городской/районный исполнительный комитет)
- Служба «Одно окно»
- Налоговая инспекция (ИМНС)
- ОГиМ (отдел гражданства и миграции)
- Поликлиника
- Суд (районный/городской)
- Орган социальной защиты (управление по труду, занятости и соцзащите)
- ЖКХ / РСЦ (расчётно-справочный центр)

Выдай JSON-массив. Структура каждого объекта СТРОГО такая:
{
  "id": "<латиницей-через-дефис, напр. zags-brest-leninsky>",
  "short_name": "<короткое название>",
  "full_name": "<полное официальное название>",
  "institution_type": "<один из типов выше, точь-в-точь>",
  "region": "<область или 'г. Минск'>",
  "city": "<город>",
  "district": "<район или ''>",
  "settlement": "<населённый пункт>",
  "address": "<полный почтовый адрес>",
  "phone": "<реальный телефон в формате +375 XX XXX-XX-XX>",
  "website": "<официальный сайт>",
  "working_hours": "<часы работы>",
  "services": ["<услуга 1>", "<услуга 2>"],
  "tags": ["<тег>", "<тег>"],
  "latitude": <число>,
  "longitude": <число>,
  "is_active": true
}

ВАЖНО:
- Только реальные учреждения, реальные адреса и телефоны (с официальных сайтов
  gov.by, исполкомов, mvd.gov.by, nalog.gov.by, court.gov.by).
- Если точный телефон неизвестен — ставь телефон приёмной/канцелярии ведомства.
- institution_type должен совпадать с одним из списка ДОСЛОВНО.
- Минимум 80 учреждений, охватывая все 6 областей + Минск.
- Координаты latitude/longitude — реальные (можно из карт).

Выведи чистый JSON-массив без пояснений и markdown.
```

Сохрани как: **`data/import/institutions.json`**

- family → "Семья"
- work → "Работа"
- health → "Здоровье"
- auto → "Авто"

Сделай 4-6 проблем В КАЖДОЙ категории (всего ~35-40 проблем).

Структура каждого объекта JSON СТРОГО такая:
{
  "id": "<латиницей-через-дефис>",
  "title": "<краткий заголовок проблемы от первого лица>",
  "category": "<код категории>",

- Размер госпошлин указывай в базовых величинах ИЛИ пиши «уточняется при обращении»,
  чтобы не устареть.
- Язык простой, для обычного гражданина.

Выведи чистый JSON-массив без пояснений и markdown.

## ПРОМПТ 4 — Сценарии (многоэтапные маршруты)

```
Ты — проектировщик пошаговых маршрутов госуслуг Республики Беларусь.

Составь 6-8 комплексных жизненных сценариев (многоэтапных маршрутов).
Примеры тем: рождение ребёнка, замена паспорта по возрасту, открытие ИП,
вступление в наследство, развод, покупка квартиры, переезд и регистрация,
оформление пенсии, регистрация брака, постановка авто на учёт.

Структура каждого объекта JSON СТРОГО такая:
{
  "id": "<латиницей-через-дефис>",
  "title": "<название сценария>",
  "short_description": "<краткое описание для карточки>",
  "description": "<развёрнутое описание маршрута>",
  "category": "<Семья | Документы | Жильё и ЖКХ | Налоги | Работа | Авто | Бизнес/ИП>",
  "estimated_duration": "<напр. '1-2 месяца'>",
  "difficulty": "<Лёгкая | Средняя | Сложная>",
  "stages": [
    {
      "id": "<латиницей>",
      "title": "<название этапа>",
      "description": "<что происходит на этапе>",
      "order_index": <число, начиная с 1>
    }
  ],
  "tasks": [
    {
      "id": "<латиницей>",
      "stage_id": "<id этапа из stages>",
      "title": "<конкретная задача>",
      "deadline": "<текстовый срок, напр. 'в течение 1 месяца'>",
      "due_in_days": <число дней от старта>,
      "institution_types": ["<тип учреждения, опционально>"],
      "depends_on": ["<id задач, от которых зависит, опционально>"],
      "documents": [
        {"title": "<документ>", "required": true}
      ]
    }
  ]
}

institution_types может быть из: ЗАГС, Исполком, Служба одно окно,
Налоговая инспекция, ОГиМ, Поликлиника, Суд, Орган социальной защиты, ЖКХ / РСЦ.

ВАЖНО:
- 4-7 этапов на сценарий, 2-5 задач на этап.
- depends_on связывай логично (нельзя подать заявление до сбора документов).
- Опирайся на реальный порядок процедур в РБ.

Выведи чистый JSON-массив без пояснений и markdown.
```

Сохрани как: **`data/import/scenarios.json`**

## Подсказки по качеству промптов

- Если ChatGPT обрезает ответ — попроси «продолжи JSON с того места где остановился»
  и склей вручную.
- Для учреждений (Промпт 2) можно гонять по областям отдельно:
  «только Брестская область» → склеить массивы.
- Проси модель «не выдумывай телефоны — если не уверен, ставь телефон приёмной».
- Реальные ссылки проверяй выборочно: если 404 — пусть даёт ссылку на главную ведомства.
