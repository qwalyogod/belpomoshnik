Ты работаешь с моим дипломным проектом «Белпомощник».

Задача: доработать AI-помощника для редакторов и администраторов внутри админ-панели. Он должен помогать писать статьи, новости, карточки проблем и сценарии: создать черновик, переписать, расширить, сократить, улучшить стиль, проверить структуру, сделать заголовок, сделать summary, перевести на белорусский. Доступ только для редакторов и администраторов.

Перед началом прочитай:

* `src/backend/api/assistant.py`
* `src/backend/api/admin.py`
* `src/backend/api/articles.py`
* `reactvitemaket/src/app/components/content-editor.tsx`
* `reactvitemaket/src/app/components/desktop.tsx`
* `reactvitemaket/src/app/services/api.ts`
* `reactvitemaket/src/app/data/types.ts`
* `docs/EDITOR_PANEL_VISION.md`
* `docs/AI_GUIDE.md`
* `docs/API_CONTRACTS.md`
* `docs/SECURITY_POLICY.md`

Что сейчас есть:

* Backend endpoint `/api/admin/assistant/content`
* Роли проверяются через `require_role("content_editor")`
* Есть режимы:

  * generate
  * rewrite
  * expand
  * summarize
  * translate
* Есть fallback без Groq key
* Есть простая HTML sanitization
* В `content-editor.tsx` есть popover “ИИ-помощник”

Что нужно улучшить:

1. Интерфейс редакторского AI сейчас слишком маленький.
2. Нет preview before apply.
3. AI сразу заменяет текст, что опасно.
4. Нет режима “улучшить заголовок”.
5. Нет режима “проверить факты/риски”.
6. Нет режима “сделать структуру статьи”.
7. Нет режима “адаптировать для обычного гражданина”.
8. Нет версии “до/после”.
9. Нет сохранения редакторских AI-сессий.
10. Нет нормальных предупреждений, что AI-черновик надо проверить.
11. TypeScript mode не включает `translate`.
12. Нужно запретить доступ гражданам.

Нужно сделать backend.

1. Расширь `ContentMode`.

Добавь режимы:

* `generate`
* `rewrite`
* `expand`
* `summarize`
* `translate`
* `headline`
* `outline`
* `simplify`
* `proofread`
* `fact_check`
* `seo`
* `social`
* `neutralize`

Что делает каждый:

* `generate` — создать черновик по подсказке.
* `rewrite` — переписать текст без изменения фактов.
* `expand` — расширить, добавить структуру и пояснения.
* `summarize` — сократить summary и body.
* `translate` — перевести на белорусский.
* `headline` — предложить 5 заголовков.
* `outline` — сделать структуру материала.
* `simplify` — объяснить простым языком для гражданина.
* `proofread` — исправить ошибки и стиль.
* `fact_check` — найти места, где нужны источники или проверка.
* `seo` — улучшить заголовок/summary/tags без кликбейта.
* `social` — сделать короткий анонс.
* `neutralize` — убрать эмоциональность, сделать официально-нейтральный стиль.

2. Расширь `ContentKind`.

Backend и frontend должны согласованно поддерживать:

* `news`
* `scenario`
* `problem`
* `law`
* `guide`

Если frontend пока реально редактирует только news/scenario/problem — всё равно типы должны не конфликтовать с backend.

3. Измени response schema.

Сейчас ответ:

* `title`
* `summary`
* `body_html`
* `source`

Сделай расширенный ответ:

```json
{
  "title": "string",
  "summary": "string",
  "body_html": "string",
  "suggestions": [
    {
      "type": "warning|improvement|source_needed|headline",
      "text": "string"
    }
  ],
  "headlines": ["string"],
  "tags": ["string"],
  "source": "llm|local",
  "model": "string",
  "disclaimer": "AI-черновик требует проверки редактором."
}
```

Для совместимости frontend может игнорировать лишние поля.

4. Усиль prompt редакторского AI.

Обнови `CONTENT_SYSTEM_PROMPT`.

Правила:

* писать по-русски, если не выбран translate;
* сохранять факты;
* не придумывать номера законов, статьи, суммы, даты, телефоны, адреса;
* если источник не указан — писать “требуется проверка источника”;
* не вставлять `<script>`, `<style>`, `<iframe>`, inline event handlers;
* body_html только безопасные теги:

  * `p`
  * `ul`
  * `ol`
  * `li`
  * `strong`
  * `em`
  * `h3`
  * `h4`
  * `a`
  * `blockquote`
* не использовать markdown;
* не публиковать материал автоматически;
* для правовых тем обязательно добавлять предупреждение о необходимости проверки официального источника;
* для новостей не делать кликбейт;
* для сценариев использовать структуру:

  * что произошло;
  * что сделать сейчас;
  * какие документы;
  * куда обратиться;
  * сроки;
  * что проверить перед подачей.

5. Улучши sanitization.

Текущая `_strip_dangerous_html` удаляет опасные теги regex-ом.
Оставь её, но усили:

* удалять `javascript:`;
* удалять `data:` в href/src;
* удалять `on*` attributes;
* удалять неизвестные теги или экранировать;
* разрешить только безопасные href `http`, `https`, относительные `/...`, `mailto` при необходимости;
* не пропускать `<img>` в body_html, если editor для медиа использует отдельные поля.

6. Добавь preview-before-apply на frontend.

В `content-editor.tsx`:

* AI не должен сразу заменять текст.
* После ответа показывай модалку/панель:

  * “Предложение ИИ”
  * title
  * summary
  * body preview
  * suggestions/warnings
  * кнопки:

    * “Применить всё”
    * “Применить только summary”
    * “Применить только текст”
    * “Скопировать”
    * “Отмена”
* Если source=`local`, показывать: “Работает локальный fallback, качество ограничено”.
* Если mode=`fact_check`, не заменять текст вообще, а показать список предупреждений.

* fact_check — вернуть список “проверьте дату”, “проверьте источник”, “проверьте сумму/срок”.
