Что сейчас плохо:

1. Документы в UI пишут, что данные зашифрованы, но на деле:

   * номер документа хранится обычной строкой;
   * `issued_by`, `comment`, `custom_fields_json` хранятся открыто;
   * скан PDF сохраняется обычным файлом;
   * `/uploads/documents` смонтирован как статическая папка, и сканы можно открывать прямой ссылкой.
2. `UserDocument` во frontend не хранит нормальную информацию о скане.
3. `DocumentCardModal` показывает скан только сразу после загрузки, но не умеет нормально подгружать существующий скан после перезапуска.
4. Загрузка поддерживает только PDF, а на защите удобнее показать PDF и фото-скан.
5. Нет полноценного demo seed для всех пользователей.

Нужно сделать backend-часть.

1. Добавь сервис шифрования, например `src/backend/security/document_crypto.py` или `src/backend/services/document_crypto.py`.

Требования к шифрованию:

* Используй уже установленный пакет `cryptography`.
* Для файлов и чувствительных полей используй нормальное симметричное шифрование, например `Fernet` или AES-GCM.
* Ключ брать из env-переменной `BELPOMOSHNIK_DOCUMENT_MASTER_KEY`.
* Добавь понятную ошибку при отсутствии ключа в обычном запуске.
* Для тестов разреши безопасный тестовый ключ через monkeypatch/env.
* Добавь в `.env.example` пример переменной и команду генерации ключа:
  `python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"`
* Не логируй ключи и расшифрованные данные.

2. Переработай модель личных документов.

Сохрани публичные/служебные поля, которые нужны для списка и напоминаний:

Зашифруй at-rest:

Можно сделать через новые колонки:

* `scan_encrypted_path`

Старые поля сразу не удаляй. Сделай мягкую совместимость:

* при чтении, если encrypted-поле есть — расшифровывай;
* если encrypted-поля нет, но старое plain-поле заполнено — возвращай старое значение;
* при создании/обновлении записывай уже в encrypted-поля;
* plain-поля после сохранения можно оставлять пустыми или сохранять только для backward compatibility, но чувствительные данные не должны дублироваться открыто.

Добавь миграцию в `src/backend/migrations/00xx_document_encryption.sql`.
Обнови `models.py`.
Обнови схемы/response в `api/user.py`.

3. Переработай scan upload/download.

Текущий подход с `/uploads/documents` как StaticFiles нужно убрать или обезвредить:

* Не отдавай личные документы через static mount.
* Сканы должны лежать вне публичной папки, например `data/secure_documents/...`.
* В `src/backend/app.py` убери отдельный static mount `/uploads/documents` или оставь только если он больше не указывает на личные документы.
* Endpoint `GET /api/user/documents/{doc_id}/scan` должен:

  * проверять JWT;
  * проверять владельца;
  * читать encrypted файл;
  * расшифровывать в памяти;
  * отдавать `StreamingResponse` или `Response/FileResponse` с правильным `Content-Type`;
  * выставлять безопасный filename.
* Endpoint `POST /api/user/documents/{doc_id}/scan` должен:

  * принимать PDF, JPEG, PNG, WEBP;
  * проверять MIME и magic bytes;
  * ограничивать размер, например 8 МБ;
  * шифровать файл перед записью;
  * сохранять metadata в БД;
  * удалять старый encrypted файл при замене;
  * возвращать не публичный URL, а metadata: `has_scan`, `scan_size`, `scan_mime_type`, `scan_uploaded_at`, `download_url`.
* Endpoint `DELETE /api/user/documents/{doc_id}/scan` должен удалять encrypted файл и очищать metadata.

6. Добавь демо-наполнение документов для всех существующих пользователей.

Скрипт должен:

* пройти по всем users, кроме гостя если он существует;
* для каждого пользователя создать набор demo документов, если таких ещё нет;
* значения генерировать псевдослучайно, но детерминированно от `user.id` или `user.email`, чтобы повторный запуск не плодил хаос;
* не использовать реальные номера документов;
* помечать данные как demo.

Набор документов:

* Паспорт гражданина Республики Беларусь
* ID-карта / биометрический документ
* Водительское удостоверение, если `has_car=True`

* Документ на жильё / договор найма, если `owns_property=True` или `is_renter=True`
* Медицинская справка / медкнижка
* Налоговое извещение / УНП физического лица

Для части документов сгенерируй фейковый скан:

* можно создать минимальный PDF с текстом “DEMO SCAN — NOT A REAL DOCUMENT”;

Скрипт должен быть безопасным:

* не перезаписывать документы, если уже есть документ с таким `doc_type/title` для пользователя;
