Ты работаешь с моим дипломным проектом «Белпомощник».

Задача: полностью доработать frontend AI-помощника для обычных пользователей. Нужно сделать удобный интерфейс почти как ChatGPT: адаптивный desktop/tablet/mobile, история диалогов, ссылки на материалы проекта, кнопки действий, добавление сценария в “Мои ситуации” прямо из чата, настройки Groq key.

Перед началом прочитай:

* `reactvitemaket/src/app/components/extra-screens.tsx`
* `reactvitemaket/src/app/components/content-editor.tsx`
* `reactvitemaket/src/app/services/api.ts`
* `reactvitemaket/src/app/data/types.ts`
* `reactvitemaket/src/app/data/store.tsx`
* `reactvitemaket/src/app/pages.tsx`

* `src/backend/api/assistant.py`
* `docs/AI_GUIDE.md`
* `docs/DESIGN_SYSTEM.md`
* `docs/API_CONTRACTS.md`

Важное:

* Не ломай текущий ShellContext и кнопку открытия помощника.
* Не переписывай всё приложение.
* Дизайн должен быть современным, чистым и удобным.
* Интерфейс должен нормально работать на desktop, tablet и mobile.
* На mobile чат должен открываться почти на весь экран.
* На desktop чат может быть боковой панелью или большим floating modal.
* Не показывай пользователю технические JSON/error stack.
* Не говори “ИИ точно знает закон”. Формулируй осторожно.

Что сейчас плохо:

1. `AssistantPanel` в `extra-screens.tsx` — маленькое floating-окно, больше похожее на виджет, чем на полноценный чат.
2. Нет списка диалогов.
3. Нет сохранения истории на аккаунте.
4. Нет streaming UI, хотя backend уже имеет `/api/assistant/chat/stream`.
5. Нет кнопок действий из ответа.
6. Пользователь может только открыть ссылку, но не может прямо из чата добавить сценарий в “Мои ситуации”.
7. Нет настройки Groq key.
8. Нет нормального состояния “AI не настроен”.
9. Нет starter prompts.
10. Нет удобного empty state.
11. Нет отдельного UI для sources/links/actions.
12. Нет нормальной обработки ошибок rate limit / invalid key.

Нужно сделать новый frontend.

2. Типы.

В `assistantTypes.ts` добавь:

```ts
export type AssistantRole = "user" | "assistant";

export interface AssistantLink {
  title: string;
  path: string;
  kind: "scenario" | "problem" | "authority" | "document" | "news" | "law" | "section" | string;
  snippet?: string;
}

export interface AssistantAction {
  type: "create_user_situation" | "open_link" | "search_catalog";
  label: string;
  scenarioId?: string;
  title?: string;
  path?: string;
  disabled?: boolean;
  completed?: boolean;
}

export interface AssistantSource {
  kind: string;
  title: string;

}

export interface AssistantMessage {
  id: string;
  role: AssistantRole;
  text: string;
  createdAt: string;
  links?: AssistantLink[];
  actions?: AssistantAction[];
  sources?: AssistantSource[];
  source?: "llm" | "local";

}

export interface AssistantSession {
  id: string;
  title: string;

  userKeyConfigured: boolean;
  maskedKey?: string;
  model?: string;

}
```

4. Новый UX обычного AI-помощника.

Сделай интерфейс примерно как ChatGPT:

Desktop:

* слева узкая панель истории диалогов:

  * “Новый чат”
  * список последних чатов
  * поиск по чатам, если быстро
  * кнопка настроек AI
* справа основная область:

  * header: “AI-помощник Белпомощника”
  * subtitle: “Подсказывает по материалам проекта и помогает найти нужный сценарий”
  * message list
  * composer внизу

Tablet:

* история скрывается в drawer;
* кнопка “История” в header.

Mobile:

* полноэкранный bottom sheet / modal;
* header закреплён сверху;
* composer закреплён снизу;
* история открывается отдельной панелью;
* кнопки должны быть крупными;
* textarea не должна ломать верстку при клавиатуре.

5. Empty state.

Когда чат новый, покажи:

* крупный заголовок: “Чем помочь?”
* пояснение: “Опишите проблему обычными словами — я найду подходящие материалы в Белпомощнике.”
* starter prompts:

  * “Я потерял паспорт, что делать?”
  * “Куда обратиться по ЖКХ?”
  * “Как оформить ситуацию по рождению ребёнка?”
  * “Какие документы нужны для водительского удостоверения?”
  * “Что делать с налогом на недвижимость?”
  * “Найди ближайший сценарий по моей проблеме”
* если пользователь гость:

  * “Для сохранения истории и добавления ситуаций нужен вход.”
  * кнопки “Войти” и “Продолжить без истории”.

6. Provider status.

В header или empty state показывай аккуратный статус:

* если `effectiveMode=server_key`: “AI включён”
* если `effectiveMode=user_key`: “AI работает с вашим ключом Groq”
* если `effectiveMode=local_fallback`: “AI-ответы ограничены: добавьте Groq key в настройках”

Если нет ключа:

* не блокируй полностью;
* давай local fallback;
* показывай CTA “Добавить Groq key”.

7. Настройки Groq key.

Создай `AssistantSettingsModal`.

Поля:

* masked key, если есть;
* input для нового Groq key;
* model select/input:

  * значение из backend;
  * можно оставить text input, если не хочешь поддерживать список моделей;
* кнопки:

  * “Сохранить ключ”
  * “Проверить”
  * “Удалить ключ”
* пояснение:

  * “Ключ хранится на сервере в зашифрованном виде.”
  * “Он нужен только если в проекте не настроен общий серверный ключ.”
  * “Полный ключ после сохранения не показывается.”

Ошибки:

* неправильный ключ;
* превышен лимит;
* сервер недоступен;
* ключ не сохранён.

8. История чатов.

При открытии помощника:

* если пользователь авторизован:

  * загрузить список сессий;
  * открыть последнюю активную сессию или показать новый чат;
* если гость:

  * использовать временный local state без сохранения.

Функции:

* новый чат;
* открыть старый чат;
* переименовать чат;
* удалить чат;
* после отправки первого сообщения создать session, если её ещё нет;
* сообщения подгружать из backend.

9. Отправка сообщений.

Поведение:

* Enter отправляет;
* Shift+Enter — новая строка;
* кнопка отправки disabled, если пусто;
* после отправки:

  * сообщение пользователя сразу появляется;
  * появляется typing indicator;
  * если streaming доступен — показывать ответ постепенно;
  * если streaming сломался — fallback на обычный POST;
  * после ответа от backend сохранить links/actions/sources.

Если backend вернул `source=local`, показывать маленькую пометку:
“Локальный ответ — AI-провайдер недоступен”.

* получить server id;
* корректно создать задачи;
* сохранить в аккаунте.

После успешного action:

* обнови `situations` в store;
* или вызови reload user situations;
* или адаптируй returned situation через `adaptUserSituation`.
