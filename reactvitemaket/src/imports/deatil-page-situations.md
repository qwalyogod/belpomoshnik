Страница: «Детальная страница ситуации»
Route: /situation-detail/:id

Назначение:
Пользователь выполняет личный пошаговый план.

Верх страницы:

- Кнопка назад.
- Название ситуации.
- Статус.
- Общий прогресс.
- Выполнено задач: N из M.
- Ближайший срок.
- Кнопка «Добавить задачу».
- Кнопка «Редактировать ситуацию».

Блок подсказки:
«Отмечайте задачи выполненными — прогресс обновится автоматически».

Структура задач:
Задачи должны быть сгруппированы по этапам.

Пример:
Этап 1. Получение медицинских документов

- checkbox: Получить медицинскую справку о рождении ребёнка.
- checkbox: Проверить правильность данных.

Этап 2. Подготовка документов родителей

- checkbox: Подготовить паспорт матери.
- checkbox: Подготовить паспорт отца.
- checkbox: Подготовить свидетельство о браке.

Для каждой задачи:

- Название.
- Описание.
- Статус.
- Срок.
- Документы.
- Куда обращаться.
- Чекбокс выполнения.
- Если задача заблокирована — показать замок и текст:
  «Сначала выполните: Получить свидетельство о рождении».
- Кнопки редактирования и удаления задачи.

Визуальные состояния:

- Выполнена — спокойный зелёный статус.
- Просрочена — мягкое предупреждение.
- Заблокирована — серый стиль с иконкой замка.
- Активная — обычная карточка.

Блок «Документы по ситуации»:

- Список документов, которые нужны по задачам.
- Уникальные документы без дублей.
- Обязательный / необязательный.

Mobile:

- Все этапы accordion.
- Чекбоксы крупные.
- Действия по задаче через меню «...».
