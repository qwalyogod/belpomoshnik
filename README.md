# Белпомощник

«Белпомощник» - кроссплатформенное мобильное приложение на Flet/Python для информационной поддержки граждан Республики Беларусь.

Пользователь выбирает или описывает жизненную проблему, а приложение показывает понятный пошаговый план действий: что делать, какие документы нужны, какие сроки учитывать, куда обращаться и каких ошибок избегать.

Текущий статус: первая демонстрационная бета. Рабочий интерфейс сейчас работает на Flet, основные пользовательские сценарии собраны в цельный MVP, а для админ-панели и сценариев добавлена backend-основа (FastAPI + SQLAlchemy + SQLite для MVP). Авторизация реализована как локальный MVP без JWT/RBAC.

Новый UI-курс: пользовательский frontend постепенно переносится в `reactvitemaket/` на Vite/React/TypeScript. Flet-версия остаётся рабочим baseline до тех пор, пока React/Vite-версия не повторит основной функционал. План миграции зафиксирован в `docs/REACT_MIGRATION_PLAN.md`, аудит макета — в `docs/REACT_MIGRATION_AUDIT.md`.

## Технологии

- Python.
- Flet.
- Vite + React + TypeScript (frontend-кандидат в `reactvitemaket/`).
- FastAPI (backend API для сценариев и админ-раздела).
- SQLAlchemy + SQLite (MVP база сценариев).
- Markdown-документация.
- Моковые данные и seed-данные сценариев.

- автоматический расчет прогресса ситуации по выполненным задачам;
- зависимости шагов внутри сценариев: зависимая задача блокируется, пока не выполнены предыдущие шаги;
- локальный справочник госучреждений и подбор подходящих организаций по городу/региону пользователя;

- админ-панель показывает права, доступные разделы, демо-пользователей и первые записи аудита;
- редактор в админ-панели может локально создать, отредактировать, опубликовать, вернуть в черновик и удалить закон-апдейт;
- закон-апдейты поддерживают приоритет, действие для пользователя, официальный источник, связанные сценарии и блок «Важное для вас»;
- редактор может локально вести справочник официальных источников: добавить, отредактировать, удалить и отметить источник активным или требующим проверки;
- группировка задач ситуации по этапам и фильтры: все, доступные, заблокированные, выполненные, просроченные;
- детальная ситуация показывает документы, которые нужны для задач;
- главный экран-дашборд: активные ситуации, ближайшие задачи, просрочки и документы к подготовке;
- ручные задачи внутри ситуации поддерживают дату срока, чтобы попадать в ближайшие и просроченные задачи;
- ручные задачи внутри ситуации поддерживают документы, чтобы попадать в блок подготовки документов;
- локальные уведомления автоматически создаются по ближайшим и просроченным задачам;
- iPhone-ready layout: общий SafeArea-wrapper, защита от notch/Dynamic Island и нижней home indicator-зоны, безопасная нижняя навигация;
- backend API `/api/...` для проблем, сценариев, шагов, документов и организаций;
- миграция и seed MVP-сценария «Рождение ребёнка».
- локальное сохранение пользовательского MVP-состояния в `data/app_state.json` и backup в `data/app_state.backup.json`.
- демо-сброс в профиле для повторного показа руководителю.

## Подготовка окружения

Виртуальное окружение уже создано в папке `.venv`.

Создать окружение заново:

```bash
python -m venv .venv
```

Если в системе нет команды `python`, используйте:

```bash
python3 -m venv .venv
```

Активировать окружение на macOS/Linux:

```bash
source .venv/bin/activate
```

Активировать окружение на Windows:

```bat
.venv\Scripts\activate
```

Установить зависимости:

```bash
pip install -r requirements.txt
```

## Запуск

После активации окружения:

```bash
flet run src/main.py
```

Без активации окружения:

```bash
.venv/bin/flet run src/main.py
```

Проверка web-превью на локальном ПК:

```bash
.venv/bin/flet run --web src/main.py
```

Для iPhone-проверки использовать Flet App / QR-сценарий запуска и смотреть размеры 375, 393 и 430 px. Android-адаптация выделена в отдельный будущий этап.

Запуск на подключенное iOS-устройство, если Flet CLI и устройство уже настроены:

```bash
.venv/bin/flet run --ios src/main.py
```

## React/Vite frontend-кандидат

React/Vite-макет находится в папке `reactvitemaket/`. Это новый frontend-кандидат, который постепенно будет подключаться к backend API.

Установка зависимостей:

```bash
cd reactvitemaket
pnpm install
```

Проверка сборки:

```bash
pnpm build
```

Запуск локального frontend-превью:

```bash
pnpm dev --host 127.0.0.1 --port 8560
```

Если backend запущен на другом адресе, создайте `.env` по примеру:

```bash
cp .env.example .env
```

и настройте `VITE_API_BASE_URL`.

## Backend (API и БД)

Инициализация базы данных:

```bash
PYTHONPATH=src python -m backend.scripts.init_db
```

Заполнение MVP-данными («Рождение ребёнка»):

```bash
PYTHONPATH=src python -m backend.scripts.seed_db
```

Запуск API:

```bash
PYTHONPATH=src uvicorn backend.app:app --host 127.0.0.1 --port 8060
```

## Локальное состояние

Для дипломного MVP пользовательские действия сохраняются локально в файле:

```text
data/app_state.json
```

Туда попадают локальные аккаунты, профиль, настройки, добавленные документы, ситуации, задачи, прочитанные уведомления и сохраненные карточки. Перед сохранением создается `data/app_state.backup.json`; если основной JSON поврежден, приложение пробует восстановиться из backup. Это демо-хранилище, а не production-авторизация.

Личные документы хранятся локально в структуре `personal_documents`. Настройки приватности лежат в `privacy_settings`, а выбранные через FilePicker файлы/сканы копируются в `data/private_uploads/`, если Flet отдаёт локальный путь к файлу. Эта папка, `data/app_state.json` и `data/app_state.backup.json` исключены из Git.

Важно: в демонстрационной версии данные не шифруются и не отправляются на сервер. Для production-версии нужны шифрование, серверная защита, контроль доступа, политика хранения персональных данных и аудит действий.
